Une API — un restaurant qui sert des donnees.

Une API (Application Programming Interface) c'est un serveur qui expose des donnees via des URLs. Tu fais une requete HTTP vers une URL, le serveur repond avec des donnees en JSON. Exactement comme commander dans un restaurant — tu commandes (requete), le serveur apporte (reponse JSON).

Exemples concrets : CoinGecko expose les prix de toutes les cryptos. OpenWeatherMap expose la meteo. GitHub expose les repos. Twitter expose les tweets. Toutes ces apps que tu utilises exposent des APIs.

HTTP — Les bases des requetes
// Une requete HTTP c'est :
// URL : ou on envoie la requete
// Methode : GET (lire), POST (creer), PUT (modifier), DELETE (supprimer)
// Headers : metadonnees (auth token, type de contenu...)
// Body : donnees envoyees (pour POST/PUT)

// Exemple : API CoinGecko (publique, sans auth)
// GET https://api.coingecko.com/api/v3/simple/price
//   ?ids=bitcoin,ethereum
//   &vs_currencies=usd
//   &include_24hr_change=true

// Reponse JSON :
// {
//   "bitcoin": { "usd": 65000, "usd_24h_change": 2.3 },
//   "ethereum": { "usd": 3200, "usd_24h_change": -1.1 }
// }

// Status codes HTTP
// 200 OK — succes
// 201 Created — cree avec succes
// 400 Bad Request — requete invalide
// 401 Unauthorized — pas authentifie
// 404 Not Found — ressource inexistante
// 500 Server Error — erreur cote serveur

REST vs GraphQL
La majorite des APIs publiques sont des APIs REST : des URLs structurees avec des methodes HTTP. GraphQL est une alternative ou tu specifies exactement quelles donnees tu veux dans la requete. GitHub, Twitter et Shopify proposent les deux. Pour debuter, les APIs REST sont plus simples — tu appelles une URL et tu recois du JSON.
fetch et async/await — charger des donnees de facon asynchrone.

fetch() est asynchrone — ca prend du temps de faire une requete reseau. JavaScript ne bloque pas en attendant. async/await permet d'ecrire ce code asynchrone comme s'il etait synchrone, de facon lisible.

JavaScript — fetch et async/await
// fetch retourne une Promise
// async/await simplifie la gestion des Promises

async function getPrixBTC() {
  // await attend que fetch soit termine
  const response = await fetch(
    'https://api.coingecko.com/api/v3/simple/price?ids=bitcoin&vs_currencies=usd'
  );

  // Convertir la reponse en JSON
  const data = await response.json();

  // Acceder aux donnees
  const prix = data.bitcoin.usd;
  console.log(`BTC : ${prix} $`);
  return prix;
}

// Appeler la fonction
getPrixBTC();

// Avec plusieurs cryptos
async function getMarket() {
  const res = await fetch(
    'https://api.coingecko.com/api/v3/simple/price' +
    '?ids=bitcoin,ethereum,solana' +
    '&vs_currencies=usd' +
    '&include_24hr_change=true'
  );
  const data = await res.json();

  // data = { bitcoin: {usd: 65000, usd_24h_change: 2.3}, ... }
  for (const [coin, info] of Object.entries(data)) {
    console.log(`${coin}: ${info.usd} $ (${info.usd_24h_change.toFixed(2)}%)`);
  }
}
await = "attends que ca soit fini"await stoppe l'execution de la fonction async jusqu'a ce que la Promise soit resolue. Mais ca ne bloque PAS le reste du JS — les autres evenements continuent de fonctionner. C'est la magie de l'asynchrone : tu attends la reponse reseau sans bloquer l'interface utilisateur.
Gestion d'erreurs — try/catch et les cas qui foires.

Les requetes reseau peuvent echouer — pas de connexion, API down, URL incorrecte, rate limit. Il faut toujours gerer les erreurs pour ne pas casser l'interface utilisateur et donner un feedback comprehensible.

JavaScript — try/catch avec fetch
async function getPrix(coin) {
  try {
    // Afficher un loader
    statusEl.textContent = 'Chargement...';

    const res = await fetch(`https://api.coingecko.com/api/v3/simple/price?ids=${coin}&vs_currencies=usd`);

    // Verifier le status HTTP
    if (!res.ok) {
      throw new Error(`Erreur HTTP : ${res.status}`);
    }

    const data = await res.json();
    statusEl.textContent = 'OK';
    return data;

  } catch (error) {
    // Erreur reseau OU erreur HTTP
    console.error('Erreur fetch :', error.message);
    statusEl.textContent = 'Erreur de connexion';
    return null; // Valeur par defaut
  }
}

// fetch ne rejette PAS sur les erreurs HTTP (404, 500)
// Il faut verifier res.ok (true si status 200-299)
// Il rejette seulement sur les erreurs reseau (pas de connexion)

// Loading state pattern
async function chargerDonnees() {
  btn.disabled = true;
  btn.textContent = 'Chargement...';
  try {
    const data = await fetchData();
    afficher(data);
  } finally {
    btn.disabled = false;
    btn.textContent = 'Rafraichir';
  }
}
⚠️
res.ok — le piege classique
fetch() ne rejette pas sur les erreurs HTTP. Si le serveur repond 404 ou 500, fetch() resout quand meme la Promise (pas d'erreur lancee). Tu dois verifier res.ok (true si status 200-299) ou res.status. Ce comportement surprend tous les debutants. Toujours verifier if(!res.ok) throw new Error(res.status) avant d'appeler res.json().

JavaScript — Le code exact de cette demo
async function fetchPrix() {
  const status = document.querySelector('#fetch-status');
  status.textContent = 'Chargement...';

  try {
    const res = await fetch(
      'https://api.coingecko.com/api/v3/simple/price' +
      '?ids=bitcoin,ethereum,solana,binancecoin' +
      '&vs_currencies=usd&include_24hr_change=true'
    );
    if (!res.ok) throw new Error(res.status);
    const data = await res.json();

    // Mettre a jour le DOM
    const coins = {
      bitcoin: 'btc', ethereum: 'eth',
      solana: 'sol', binancecoin: 'bnb'
    };
    for (const [id, short] of Object.entries(coins)) {
      const d = data[id];
      document.querySelector(`#fc-${short}`).textContent =
        `$${d.usd.toLocaleString()}`;
      const varEl = document.querySelector(`#fc-${short}-var`);
      const v = d.usd_24h_change.toFixed(2);
      varEl.textContent = (v > 0 ? '+' : '') + v + '%';
      varEl.className = 'fc-var ' + (v > 0 ? 'up' : 'down');
    }
    status.textContent = `Mis a jour : ${new Date().toLocaleTimeString()}`;
  } catch (err) {
    status.textContent = 'Erreur : ' + err.message;
  }
}
