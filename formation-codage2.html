Module 2 — HTML
La structure d'une page HTML.
Chaque page web correctement écrite suit la même structure. Comprendre cette structure c'est comprendre comment le navigateur lit et interprète ton code — et pourquoi certaines choses marchent ou ne marchent pas.

Exercice 🎯
Un fichier HTML c'est un fichier texte. Rien de plus.

Ouvre le Bloc-notes sur Windows, TextEdit sur Mac, ou n'importe quel éditeur de texte. Écris dedans. Sauvegarde en .html. Ouvre dans Chrome. Tu viens de créer une page web.

Pas de logiciel spécial, pas d'installation compliquée. C'est du texte que le navigateur sait lire. Ce qui change c'est ce que tu mets dedans — et il y a une structure précise à respecter.

Tout fichier HTML bien écrit ressemble à ça :

Clique sur chaque section pour la voir en détail ↓
<!DOCTYPE html>
Déclaration Tap pour voir
→ Toujours en première ligne. Dit au navigateur "ce qui suit c'est du HTML5". Sans ça le navigateur entre en "mode quirks" et peut se comporter bizarrement.
<html lang="fr">
Racine Tap pour voir
→ Contient TOUT le reste. La balise la plus englobante. L'attribut lang="fr" dit que la page est en français — utile pour les lecteurs d'écran et les moteurs de recherche. Se ferme tout à la fin avec </html>.
  <head> ... </head>
Invisible Tap pour voir
→ Ce que l'utilisateur ne voit PAS directement. Le titre de l'onglet, les métadonnées, les liens vers le CSS, le charset... Tout ce que le navigateur et les moteurs de recherche ont besoin de savoir sur ta page.
  <body> ... </body>
Visible Tap pour voir
→ Tout ce que l'utilisateur VOIT sur la page. Les titres, les textes, les images, les boutons, les formulaires... Si ça s'affiche à l'écran, ça va dans le body. C'est là que tu passes 90% de ton temps.

Ce n'est pas une convention inventée au hasard. C'est le standard international défini par le W3C — l'organisation qui régule le web depuis 1994. Tous les navigateurs du monde respectent ce standard.

Ça veut dire que si tu respectes cette structure, ta page fonctionnera sur Chrome, Firefox, Safari, Edge, sur PC, Mac, iPhone, Android. Partout.

🌍
Le W3C
Le World Wide Web Consortium, fondé par Tim Berners-Lee (l'inventeur du web), définit les standards HTML depuis 1994. Sans eux, chaque navigateur pourrait interpréter le HTML différemment et tout le web serait cassé. C'est grâce à eux qu'une page que tu codes aujourd'hui fonctionnera encore dans 10 ans.
La première ligne de toute page web. Le DOCTYPE.

Avant même la balise HTML, avant tout le reste, tu mets une seule ligne :

HTML — Ligne 1 de tout fichier web
<!DOCTYPE html>
<!-- C'est tout. Une ligne. Toujours la même. -->

Cette ligne dit au navigateur : "Ce fichier est du HTML5, la version moderne du HTML." Sans cette ligne, le navigateur entre dans ce qu'on appelle le "quirks mode" — un mode de compatibilité avec les vieux sites des années 90 où les règles étaient différentes. En quirks mode, ton CSS peut se comporter de façon imprévisible.

Bonne nouvelle : tu n'as jamais à y penser. Tu la mets, tu passes à la suite. Elle ne change jamais.

Pourquoi c'est écrit comme ça Le DOCTYPE a existé dans des versions plus complexes avant HTML5. En HTML4 il ressemblait à ça : <!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN" "http://www.w3.org/TR/html4/strict.dtd">. Les créateurs de HTML5 ont décidé de simplifier ça en juste <!DOCTYPE html>. Merci à eux.

Le <head> — ce que l'utilisateur ne voit pas, mais qui change tout.

Tu regardes une page web et tu vois le contenu. Mais derrière, il y a des informations que le navigateur et les moteurs de recherche utilisent sans jamais les afficher directement à l'écran. Tout ça vit dans le <head>.

👁<head> — Invisible
  • Le titre de l'onglet du navigateur
  • L'encodage des caractères (é, à, ç...)
  • La description pour Google
  • Le lien vers ton fichier CSS
  • Les instructions pour mobile
  • L'icône de l'onglet (favicon)
🖥<body> — Visible
  • Les titres et les textes
  • Les images
  • Les boutons
  • Les menus de navigation
  • Les formulaires
  • Tout ce que tu vois à l'écran

Voici les balises les plus importantes dans un <head> :

<meta charset="UTF-8">
Encodage des caractères — INDISPENSABLE
Sans ça, tes é, à, ç, ñ, 中文, العربية vont s'afficher en caractères bizarres. UTF-8 supporte tous les caractères de toutes les langues du monde. Toujours mettre en premier dans le head.
<meta name="viewport"...>
Responsive mobile — INDISPENSABLE
Sans ça, ton site va s'afficher en version "desktop zoomée" sur mobile — illisible. Cette balise dit au téléphone d'adapter la page à sa taille d'écran. En 2024, plus de 60% du trafic web vient du mobile.
<title>...</title>
Le titre de la page
Ce qui s'affiche dans l'onglet du navigateur et dans les résultats Google. "Kora — L'information vérifiée" ou "Accueil — Amazon". Important pour le référencement SEO.
<meta name="description">
La description Google
Le texte qui apparaît sous le titre dans les résultats de recherche Google. Ne s'affiche pas sur la page mais crucial pour le SEO et pour donner envie aux gens de cliquer.
<link rel="stylesheet">
Lien vers le fichier CSS
C'est ici que tu dis au navigateur "pour styliser cette page, va lire ce fichier CSS". Sans ça, ta page sera du HTML brut sans mise en forme.
HTML — Un head complet et correct
<head>
  <!-- 1. Encodage — TOUJOURS en premier -->
  <meta charset="UTF-8">

  <!-- 2. Responsive mobile -->
  <meta name="viewport" content="width=device-width, initial-scale=1.0">

  <!-- 3. Titre de l'onglet et de Google -->
  <title>Kora — L'information vérifiée</title>

  <!-- 4. Description pour Google -->
  <meta name="description" content="Kora agrège, vérifie et analyse les news en temps réel.">

  <!-- 5. Lien vers le CSS -->
  <link rel="stylesheet" href="kora.css">
</head>
Ce que Google voit Quand Google indexe une page, il lit le <head> en premier. Le title et la meta description décident directement de comment ta page apparaît dans les résultats de recherche. Un bon head = un meilleur référencement. C'est ce qu'on appelle le SEO on-page.
🔍
Teste maintenant sur Google
Va sur Google, cherche n'importe quoi. Le titre en bleu cliquable, c'est le contenu de la balise <title> du site. Le texte en gris en dessous, c'est la <meta name="description">. Des millions de développeurs ont soigneusement écrit ces textes pour que tu cliques dessus.
Le <body> — tout ce que l'utilisateur voit.

Si le <head> c'est les coulisses, le <body> c'est la scène. Tout ce qui s'affiche à l'écran vit ici. C'est là où tu vas passer la majorité de ton temps quand tu codes une page.

Il n'y a qu'un seul <body> par page, et tout ton contenu se met dedans — dans l'ordre dans lequel tu veux qu'il apparaisse de haut en bas.

HTML — Une page complète de A à Z
<!DOCTYPE html>
<html lang="fr">

<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Ma première vraie page</title>
  <meta name="description" content="Une page créée par moi, de zéro.">
</head>

<body>

  <h1>Bienvenue sur ma page</h1>

  <p>Je m'appelle Val. Je suis en train d'apprendre le code.</p>

  <h2>Ce que je construis</h2>

  <p>Je travaille sur <strong>Kora</strong>, une plateforme d'information.</p>

</body>

</html>
Ce que le navigateur affiche — uniquement le contenu du body

Bienvenue sur ma page

Je m'appelle Val. Je suis en train d'apprendre le code.

Ce que je construis

Je travaille sur Kora, une plateforme d'information.

Regarde bien le code. Tu peux voir que les balises s'imbriquent — les unes dans les autres. Le <body> est dans le <html>. Les <h1> et <p> sont dans le <body>. Le <strong> est dans le <p>.

C'est comme des poupées russes. Et la règle c'est que tu dois toujours fermer dans l'ordre inverse où tu as ouvert. Si tu ouvres A puis B, tu fermes B puis A. Jamais l'inverse.

<!DOCTYPE html>
Ligne 1 — toujours. Déclare que c'est du HTML5. Le navigateur entre en mode standard.
<html lang="fr">
Ouvre le document HTML. Lang="fr" précise la langue — important pour l'accessibilité et Google.
<head> ... </head>
Zone invisible. Charset, viewport, title, meta description, liens CSS. Tout ce que le navigateur a besoin de savoir.
<body> ... </body>
Zone visible. Tout le contenu affiché à l'écran. H1, p, images, boutons... tout ici.
</html>
Dernière ligne — toujours. Ferme la balise html ouverte au début. Tout le document est entre <html> et </html>.

La règle d'or de l'imbrication Les balises doivent être fermées dans l'ordre inverse de leur ouverture. Si tu ouvres <p> puis <strong>, tu fermes </strong> puis </p>. Jamais </p></strong> — c'est comme fermer la boîte avant d'y mettre le couvercle.
Tu connais maintenant la structure complète. À toi de la construire.

C'est le moment de tout assembler — DOCTYPE, html, head avec les balises obligatoires, et body avec du contenu. Pas un bout de code — une vraie page complète, structurée correctement de la première à la dernière ligne.
