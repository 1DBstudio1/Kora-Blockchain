localStorage — stocker des donnees dans le navigateur.

localStorage est une API navigateur qui stocke des paires cle:valeur. Les donnees persistent apres fermeture du navigateur et rechargement de page. Limite : environ 5MB, strings seulement (pas d'objets directement).

JavaScript — localStorage
// Ecrire
localStorage.setItem('theme', 'dark');
localStorage.setItem('krt', '248');

// Lire
const theme = localStorage.getItem('theme');
// → 'dark' (string)

// Si la cle n'existe pas → null
localStorage.getItem('inexistant'); // → null

// Supprimer une cle
localStorage.removeItem('theme');

// Tout supprimer (attention !)
localStorage.clear();

// Voir toutes les cles
Object.keys(localStorage);

// Nombre d'entrees
localStorage.length;

localStorage vs sessionStorage vs cookieslocalStorage persiste indefiniment jusqu'a suppression manuelle. sessionStorage disparait a la fermeture de l'onglet. Les cookies ont une date d'expiration et sont envoyes au serveur a chaque requete (utilises pour l'auth). Pour les preferences et donnees non-sensibles cote client : localStorage. Pour les tokens d'authentification : cookies HttpOnly (plus securises).
Stocker des objets — JSON.stringify et JSON.parse.

localStorage ne stocke que des strings. Pour stocker un objet ou un tableau, il faut le convertir en string JSON avec JSON.stringify(), et le re-convertir en objet avec JSON.parse() quand on le lit.

JavaScript — Stocker des objets
const portfolio = [
  { ticker: 'BTC', quantite: 0.5 },
  { ticker: 'ETH', quantite: 2 },
];

// Sauvegarder un tableau/objet
localStorage.setItem(
  'portfolio',
  JSON.stringify(portfolio)
);
// Stocke : '[{"ticker":"BTC","quantite":0.5},...]'

// Lire un tableau/objet
const raw = localStorage.getItem('portfolio');
const data = JSON.parse(raw);
// data est maintenant un vrai tableau JS

// Pattern securise avec valeur par defaut
function lirePortfolio() {
  try {
    const raw = localStorage.getItem('portfolio');
    return raw ? JSON.parse(raw) : [];
  } catch {
    return []; // JSON.parse peut planter si donnee corrompue
  }
}

// C'est exactement comme ca que Kora sauvegarde ta progression
const prog = JSON.parse(localStorage.getItem('kora_prog') || '{}');
prog.js10 = 99;
localStorage.setItem('kora_prog', JSON.stringify(prog));

Pattern complet — une mini-app avec persistance.

Le pattern classique avec localStorage : charger les donnees au demarrage, les modifier en memoire, sauvegarder a chaque changement. L'app ci-dessous garde ton portfolio meme si tu recharges la page.

JavaScript — Pattern localStorage complet
// 1. Charger au demarrage
let portfolio = charger('mon_portfolio', []);

// 2. Fonction de chargement securisee
function charger(cle, defaut) {
  try {
    const val = localStorage.getItem(cle);
    return val ? JSON.parse(val) : defaut;
  } catch { return defaut; }
}

// 3. Fonction de sauvegarde
function sauvegarder(cle, valeur) {
  localStorage.setItem(cle, JSON.stringify(valeur));
}

// 4. Ajouter et sauvegarder
function ajouterPosition(ticker, qty) {
  portfolio.push({ ticker, qty });
  sauvegarder('mon_portfolio', portfolio);
  afficher(); // Re-render le DOM
}

// 5. Supprimer et sauvegarder
function supprimerPosition(index) {
  portfolio.splice(index, 1);
  sauvegarder('mon_portfolio', portfolio);
  afficher();
}

Ton code doit
  • Lire les preferences au chargement avec localStorage.getItem + JSON.parse
