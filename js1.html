HTML structure. CSS habille. JavaScript fait agir.

JavaScript c'est ce qui rend les pages web interactives. Sans JS, une page web est un document statique — elle s'affiche, point. Avec JS, tu peux repondre aux clics, charger des donnees, animer des elements, valider des formulaires, tout ca sans recharger la page.

Quelques exemples concrets de ce que JS fait en ce moment pendant que tu lis ces lignes : la barre de progression en haut qui s'update, le bouton "Ecouter" qui active la synthese vocale, la Kora IA qui repond a tes questions, la sauvegarde de ta progression dans localStorage.

Le trio du webHTML = squelette (structure et contenu). CSS = peau (apparence et mise en page). JavaScript = muscles et cerveau (comportement et logique). Les trois fonctionnent ensemble. JS peut lire et modifier le HTML et le CSS en temps reel — c'est ca sa vraie puissance.
⚡
Cree en 10 jours
Brendan Eich a cree JavaScript en 10 jours en 1995 pour Netscape. Il s'appelait d'abord Mocha, puis LiveScript, puis JavaScript — un nom marketing pour surfer sur la popularite de Java. Java et JavaScript n'ont aucun rapport, comme car et carpet. Aujourd'hui JavaScript est le langage le plus utilise au monde selon GitHub, et tourne dans les serveurs (Node.js), les apps mobiles (React Native), les extensions de navigateur, et meme les microscontroleurs.
Trois facons d'integrer JS — une seule est la bonne pratique.

Comme le CSS, JS peut etre integre de trois facons : fichier externe, balise script dans le HTML, ou inline sur un element. Et comme le CSS, le fichier externe est la bonne pratique en production.

HTML — Integrer JavaScript
/* Methode 1 — Fichier externe (recommande) */
<script src="script.js" defer></script>
/* defer = execute apres que le HTML est parse */
/* Mettre juste avant </body> ou dans <head> avec defer */

/* Methode 2 — Script inline */
<script>
  console.log('Bonjour Kora');
</script>

/* Methode 3 — Inline sur element (a eviter) */
<button onclick="alert('clic!')">Clic</button>
Pourquoi defer ?Sans defer, le navigateur s'arrete de parser le HTML pour executer le JS — ce qui peut ralentir l'affichage. defer dit : "parse le JS mais execute-le seulement apres que tout le HTML est charge". C'est toujours ce qu'on veut. async execute des que le fichier est telecharge, sans attendre le HTML — utile pour les scripts independants du DOM.
📁
La structure typique d'un projet
Dans un vrai projet web tu as typiquement : index.html (structure), style.css (apparence), script.js (comportement). Le HTML importe le CSS dans le head et le JS juste avant la fermeture body. Pour des projets plus grands, tu as des dossiers js/, css/, images/. Kora Academy utilise les balises script inline pour simplifier — un seul fichier HTML par module.
console.log — ton meilleur ami en JavaScript.

console.log affiche un message dans la console du navigateur. C'est le premier outil de tout developpeur JS. Tu l'utilises pour debugger, verifier des valeurs, comprendre ce qui se passe dans ton code.

La console est visible dans les DevTools de ton navigateur — F12 ou clic droit → Inspecter → Console.

JavaScript — console
// Afficher du texte
console.log('Bonjour Kora');

// Afficher des nombres
console.log(42);
console.log(3.14);

// Afficher plusieurs valeurs
console.log('Prix BTC :', 65000);

// Autres methodes de console
console.error('Une erreur !');   // Affiche en rouge
console.warn('Attention !');    // Affiche en jaune
console.table([1, 2, 3]);       // Affiche un tableau
console.clear();                 // Vide la console

console.log en production
En developpement tu mets des console.log partout pour debugger. En production (site en ligne), il faut les retirer — ils peuvent exposer des informations sensibles et ralentir legerelement les perfs. Des outils comme ESLint peuvent t'avertir si tu oublies des console.log. Pour les logs permanents (erreurs importantes), utilise des services comme Sentry qui capturent les erreurs en production.
Les DevTools — la console de ton navigateur.

Les DevTools (outils developpeur) sont integres dans tous les navigateurs modernes. C'est ton environnement de travail principal quand tu developpes en JS. La console est l'onglet le plus utilise — tu peux y taper du JS directement et voir le resultat instantanement.

DevTools — Comment acceder et utiliser
// Ouvrir les DevTools
// Chrome/Edge : F12 ou Ctrl+Shift+I (Cmd+Option+I sur Mac)
// Firefox : F12
// Ou : clic droit sur la page → Inspecter

// Dans l'onglet Console, tu peux taper directement :
2 + 2          // → 4
'Kora'.toUpperCase()  // → 'KORA'
alert('Bonjour')  // → Ouvre une popup

// Les DevTools ont aussi :
// Elements : voir et modifier le HTML/CSS en live
// Network : voir les requetes HTTP
// Sources : debugger le JS avec des breakpoints
// Application : voir localStorage, cookies
L'onglet ElementsDans Elements (ou Inspecteur sur Firefox), tu peux cliquer sur n'importe quel element de la page et voir son HTML et CSS. Tu peux meme les modifier en live — utile pour tester des changements CSS sans toucher aux fichiers. Les modifications disparaissent au rechargement.
🛠
Les DevTools comme terrain de jeu
La console des DevTools est le meilleur endroit pour apprendre JS rapidement. Tu peux taper du code et voir le resultat instantanement. Essaie : ouvre la console (F12) et tape document.body.style.background = 'red'. La page devient rouge. C'est la puissance de JS appliquee en direct sur une vraie page web. Tout ce que tu vois dans ce module Kora est accessible et modifiable depuis la console.
