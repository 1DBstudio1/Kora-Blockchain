Position et z-index.

Par defaut tout est en position static.

static c'est la valeur par defaut de tous les elements. Ils suivent le flux normal de la page — l'un apres l'autre. Les proprietes top, right, bottom, left n'ont aucun effet en static.

position: relative deplace un element PAR RAPPORT A SA POSITION NORMALE, sans le sortir du flux. L'espace qu'il occupait reste reserve.

Visualisation — static vs relative vs absolute

CSS — static et relative
/* static — defaut, top/left/right/bottom ignorees */
.normal {
  position: static;
}

/* relative — deplace depuis sa position normale */
.decale {
  position: relative;
  top: 10px;    /* 10px vers le bas */
  left: 20px;   /* 20px vers la droite */
  /* L'espace d'origine est CONSERVE dans le flux */
}

/* relative sert surtout de contexte pour absolute */
.card {
  position: relative; /* Les enfants absolute se positionnent ici */
}
La vraie utilite de relativeposition:relative est rarement utilise pour deplacer un element. Son vrai role c'est de creer un contexte de positionnement pour les elements enfants en position:absolute. Tu le verras dans la partie suivante.
position:absolute — se positionner precisement n'importe ou.

Un element absolute est retire du flux normal. Il se positionne par rapport a son ancetre le plus proche en position relative (ou au body si aucun). C'est le pattern le plus utilise pour les badges, les menus, les tooltips.

CSS — absolute en pratique
/* Pattern fondamental : parent relative + enfant absolute */
.card {
  position: relative; /* Contexte de positionnement */
}

.badge {
  position: absolute;
  top: 12px;
  right: 12px;
  /* Se positionne par rapport a .card */
}

/* Centrer un element en absolute */
.centered {
  position: absolute;
  top: 50%;
  left: 50%;
  transform: translate(-50%, -50%);
  /* Le translate corrige le decalage du point d'ancre */
}

/* Couvrir tout le parent */
.overlay {
  position: absolute;
  inset: 0; /* = top:0 right:0 bottom:0 left:0 */
  background: rgba(0,0,0,0.5);
}

Les cas d'usage de absolute
Badges de notification (le numero rouge sur une icone), menus deroulants, tooltips au survol, icones decoratives sur des cards, overlays d'image, modals. Tout ca utilise position:absolute. Le pattern parent relative + enfant absolute est l'un des plus utilises en CSS professionnel.
fixed et sticky — rester visible pendant le scroll.

fixed reste a la meme position par rapport a la fenetre du navigateur, meme quand on scrolle. sticky est un hybride — il suit le scroll jusqu'a un certain point, puis se colle a la position definie.

CSS — fixed et sticky
/* fixed — toujours visible, meme en scrollant */
.header {
  position: fixed;
  top: 0;
  left: 0;
  right: 0;
  z-index: 100;
  /* Ce header flotte au-dessus du contenu */
  /* Attention : retirer du flux, ajouter padding-top au body */
}

/* sticky — suit le scroll puis se colle */
.nav-tabs {
  position: sticky;
  top: 56px; /* Se colle quand a 56px du haut */
  z-index: 90;
}
/* Reste dans le flux ! Pas besoin de padding-top */
fixed vs stickyfixed est completement sorti du flux — le contenu doit avoir du padding-top pour ne pas etre cache derriere. sticky reste dans le flux jusqu'au moment ou il "se colle". Sur ce site, le header du module que tu lis maintenant est en position sticky.

z-index — qui est au premier plan ?

Quand des elements se superposent (ce qui arrive souvent avec position), z-index determine lequel est "devant". Plus le z-index est eleve, plus l'element est au premier plan.

CSS — z-index
/* z-index fonctionne seulement avec position != static */

/* Convention typique dans un projet */
.contenu  { z-index: 1; }
.header   { z-index: 100; }
.dropdown { z-index: 200; }
.modal    { z-index: 1000; }
.toast    { z-index: 9999; }

/* z-index ne marche que si position est definie */
.el {
  position: relative; /* Ou absolute, fixed, sticky */
  z-index: 10;
}
Stacking contextsz-index ne fonctionne qu'entre elements du meme stacking context. Quand un element a transform, opacity, filter defini, il cree un nouveau stacking context isole. C'est pourquoi parfois z-index: 9999 ne fonctionne pas — l'element est emprisonne dans un contexte parent. La solution : verifier les parents et leurs proprietes.
