Display et visibilite.
La propriete display determine comment un element se comporte dans le flux de la page. Block, inline, inline-block, none — comprendre ca c'est comprendre comment les elements s'organisent et pourquoi certains sont cote a cote et d'autres pas.

Chaque element a un display par defaut.

Les elements HTML sont block ou inline par defaut. Les elements block prennent toute la largeur disponible et forcent un retour a la ligne. Les elements inline ne prennent que la place de leur contenu et s'inserent dans le texte.

Tu peux toujours changer le display par defaut avec CSS. C'est ce qui permet de transformer un lien en bouton ou de mettre des elements cote a cote.

CSS — block vs inline
/* Elements block par defaut */
/* h1-h6, p, div, section, ul, li, form, header, nav, main... */
div { display: block; }   /* Pleine largeur, nouvelle ligne */

/* Elements inline par defaut */
/* span, a, strong, em, img, button, input... */
span { display: inline; } /* Dans le texte, width/height ignorees */

/* Changer le display */
a { display: block; }     /* Lien qui prend toute la largeur */
li { display: inline; }   /* Elements de liste cote a cote */
Attention avec inlineSur un element inline, width et height sont ignorees. Tu ne peux pas definir la taille d'un span par defaut. margin-top et margin-bottom n'ont pas d'effet non plus. Pour avoir width/height sur un element inline, tu dois passer en inline-block.
inline-block — le meilleur des deux mondes.

inline-block combine les avantages de block (width, height, padding, margin complets) et de inline (reste dans le flux du texte sans forcer de retour a la ligne). C'est parfait pour les badges, les boutons cote a cote, les elements de navigation horizontaux.

CSS — inline-block en pratique
/* Badge de categorie */
.badge {
  display: inline-block;
  padding: 4px 12px;
  border-radius: 20px;
  font-size: 12px;
  background: rgba(107,31,42,0.08);
  color: #6B1F2A;
  /* width et height fonctionnent */
  /* reste dans le flux du texte */
}

/* Boutons cote a cote */
.btn {
  display: inline-block;
  padding: 10px 20px;
  border-radius: 20px;
  text-decoration: none;
}

inline-block vs Flexbox
inline-block etait la solution principale pour mettre des elements cote a cote avant Flexbox. Aujourd'hui on utilise plutot Flexbox pour les layouts complexes — tu l'apprendras dans le module 8. Mais inline-block reste parfait pour les cas simples comme les badges et les petits groupes de boutons.
display:none et visibility:hidden — deux facons de cacher.

Cacher un element en CSS peut se faire de trois facons differentes, avec des comportements tres differents. Choisir la bonne methode depend de ce que tu veux faire.

CSS — Cacher des elements
/* display: none — INVISIBLE et ne prend plus de place */
.hidden {
  display: none;
  /* Element retire completement du flux */
  /* Les elements adjacents bougent pour remplir l'espace */
}

/* visibility: hidden — INVISIBLE mais garde sa place */
.invisible {
  visibility: hidden;
  /* Invisible mais l'espace est conserve */
  /* Les elements adjacents ne bougent pas */
}

/* opacity: 0 — TRANSPARENT mais toujours la */
.ghost {
  opacity: 0;
  /* Invisible mais toujours cliquable ! */
  /* Utile pour les transitions d'apparition */
}

/* JavaScript + display:none */
/* element.style.display = 'none'; // Cacher */
/* element.style.display = 'block'; // Afficher */
display:none en pratiqueC'est la methode la plus utilisee pour cacher/afficher dynamiquement avec JavaScript. Le menu burger de Kora Academy l'utilise : la sidebar a display:none par defaut, et passe a display:block quand tu cliques sur le bouton. Sur mobile, certains elements de navigation sont caches avec display:none et reapparaissent sur desktop via media query.
🎯
Accessibilite et display:none
Un element avec display:none est completement invisible pour les lecteurs d'ecran aussi — c'est bien pour cacher du contenu. Mais si tu veux cacher visuellement quelque chose tout en le gardant accessible aux lecteurs d'ecran (texte descriptif pour malvoyants), utilise la technique "sr-only" : position:absolute; width:1px; height:1px; overflow:hidden.
🎯Navigation horizontale avec displayExercice
Cree une barre de navigation horizontale en utilisant les proprietes display. Les liens doivent etre cote a cote, avec du padding, et un effet au survol.
Ta nav doit avoir
  • nav ul avec list-style: none pour supprimer les puces
  • nav li en display: inline-block pour les mettre cote a cote
  • nav a en display: block avec padding pour la zone cliquable
  • Un effet :hover sur les liens qui change la couleur
  • Un element avec display: none (par exemple un badge "Nouveau")
