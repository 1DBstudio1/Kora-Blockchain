Exercice 🎯
Le web c'est des milliards de pages connectees entre elles.

La balise qui rend ca possible c'est <a> — anchor, ancre. Elle transforme n'importe quel texte ou image en lien cliquable. C'est l'invention qui a cree internet tel qu'on le connait.

Tim Berners-Lee, l'inventeur du web, a dit que les hyperliens sont la caracteristique la plus importante du HTML. Avant les liens, les documents informatiques etaient isoles. Les liens les ont tous connectes.

HTML — Anatomie d'un lien
<!-- Structure de base -->
<a href="https://kora.io">Visiter Kora</a>

<!-- href = la destination (Hypertext REFerence) -->
<!-- Le texte entre les balises = ce qui s'affiche -->

<!-- Exemples reels -->
<a href="https://google.com">Rechercher</a>
<a href="https://instagram.com">Instagram</a>
<a href="https://bitcoin.org">Bitcoin.org</a>
Rendu dans le navigateur

Visiter Kora

Rechercher   Instagram   Bitcoin.org

<a>
Anchor — la balise de lien. Elle a toujours besoin d'un attribut href pour savoir ou pointer.
href="..."
Hypertext REFerence — la destination. Peut etre une URL complete, un fichier local, une section de la meme page, ou un email.
Texte du lien
Ce qui s'affiche et sur quoi l'utilisateur clique. Doit etre descriptif — pas "cliquez ici" mais "lire l'article complet".

Chiffre
Il y a plus de 130 trillions de liens sur internet selon Google. Chacun d'eux est une balise <a href="...">. La personne qui a invente la balise <a> en 1991 ne pensait pas que ca allait creer l'economie numerique mondiale qu'on connait aujourd'hui.
🎯Verification rapide
Tu veux creer un lien vers le site de Kora. Laquelle de ces syntaxes est correcte ?
Un lien peut pointer vers quatre types de destinations.

Pas seulement vers d'autres sites — vers des pages locales, des sections de la meme page, des emails, des fichiers. Chaque cas a sa syntaxe propre.

TypeSyntaxeUtilisation
Externe
href="https://..."
Autre site web. Toujours avec https://. Ajouter target="_blank" pour ouvrir dans un nouvel onglet.
Interne
href="about.html"
Autre page de ton propre site. Juste le nom du fichier — pas besoin d'URL complete.
Ancre
href="#section"
Section de la meme page. L'element cible doit avoir id="section".
Email
href="mailto:..."
Ouvre le client email de l'utilisateur avec l'adresse pre-remplie.
HTML — Les 4 types en pratique
<!-- Lien externe — ouvre dans nouvel onglet -->
<a href="https://kora.io" target="_blank" rel="noopener">
  Visiter Kora
</a>

<!-- Lien interne — autre page du meme site -->
<a href="academy.html">Aller a l'Academy</a>

<!-- Ancre — section de la meme page -->
<a href="#contact">Aller au contact</a>
<!-- ... plus bas dans la page : -->
<section id="contact">Formulaire ici</section>

<!-- Email -->
<a href="mailto:contact@kora.io">Nous contacter</a>
target="_blank" et rel="noopener"Quand tu ouvres un lien externe dans un nouvel onglet avec target="_blank", ajoute toujours rel="noopener noreferrer". Sans ca, le site cible peut acceder a la fenetre parente de ton site via JavaScript — c'est une faille de securite connue. Tous les developpeurs serieux le font systematiquement.

Le menu de navigation de Kora
Tous les liens de navigation de Kora — "Pour toi", "Journal", "Formation", "Profil" — sont des balises <a> ou des elements avec des onclick JavaScript. La difference c'est que les vrais liens <a> sont meilleurs pour le SEO et l'accessibilite. Un robot Google peut suivre un <a href> mais pas forcement un onclick.
Les images avec <img> — la balise auto-fermante.

La balise <img> est speciale — elle n'a pas de balise fermante. On dit qu'elle est auto-fermante ou "void element". Elle affiche directement une image sans contenu a entourer.

Deux attributs sont obligatoires : src (la source de l'image) et alt (le texte alternatif pour l'accessibilite et le SEO).

HTML — Images
<!-- Image locale (dans le meme dossier) -->
<img src="logo.jpg" alt="Logo Kora">

<!-- Image avec dimensions -->
<img src="photo.jpg" alt="Photo de profil"
     width="300" height="200">

<!-- Image depuis une URL externe -->
<img src="https://site.com/image.jpg"
     alt="Description de l'image">

<!-- Image cliquable = img dans un lien a -->
<a href="https://kora.io">
  <img src="logo.jpg" alt="Kora">
</a>
src="..."
Source — le chemin vers l'image. Peut etre relatif (dans le meme dossier), absolu (chemin complet), ou une URL externe.
alt="..."
Texte alternatif — s'affiche si l'image ne charge pas. Lu par les lecteurs d'ecran. Indexe par Google Images. TOUJOURS le mettre.
width / height
Dimensions en pixels. Important pour eviter le "layout shift" — la page qui saute quand l'image charge.
Formats d'images sur le webJPG — photos, compresse bien, perd un peu de qualite. PNG — logos, screenshots, transparence. WebP — format moderne, meilleur compression que JPG et PNG. SVG — icones et logos vectoriels, infini zoom sans perte. GIF — animations simples. En 2024, privilegie WebP pour les photos et SVG pour les icones.

Instagram et les images HTML
Quand tu scrolles Instagram, chaque photo est une balise <img> avec un attribut src pointant vers les serveurs de Meta. La difference avec ton code c'est que l'URL src est generee dynamiquement par JavaScript selon ton profil et tes abonnements. Mais a la base, c'est exactement la meme balise <img src="..." alt="..."> que tu viens d'apprendre.
