Le HTML donne la structure. Le CSS donne le style.

Tu as appris le HTML — les titres, les textes, les liens, les images. Tout ca c'est la structure. Mais si tu ouvres une page HTML sans CSS, tu obtiens du texte brut noir sur fond blanc, sans couleur, sans mise en page, sans rien.

Le CSS — Cascading Style Sheets — c'est le langage qui dit comment ca ressemble. Les couleurs, les polices, les tailles, les espacements, les animations... tout ca c'est le CSS.

Regarde n'importe quel grand site — Instagram, Google, Kora. Desactive le CSS dans les outils developpeur. Tu verras le HTML nu. Ce que tu vois normalement, c'est le CSS par-dessus.

La separation des responsabilitesHTML dit QUOI est sur la page. CSS dit COMMENT ca ressemble. JavaScript dit QUE FAIRE quand tu interagis. Cette separation fondamentale te permet de changer completement le design d'un site sans toucher au HTML.

CSS Zen Garden
En 2003, un site appele CSS Zen Garden a montre la puissance du CSS en publiant exactement le meme fichier HTML avec des dizaines de designs completement differents. Certains ressemblaient a des magazines de mode, d'autres a des illustrations japonaises. Meme HTML, designs radicalement differents. Ca a revolutionne la facon dont les developpeurs pensaient le web.
Tu peux ecrire du CSS de trois facons differentes.

Fichier externe, balise style dans le head, ou directement sur l'element. Chaque methode a ses avantages — mais une seule est la bonne pratique professionnelle.

Methode 1 — Fichier externe (recommande)
/* Dans le head du HTML */
<link rel="stylesheet" href="style.css">

/* Dans style.css */
h1 {
  color: #6B1F2A;
  font-size: 32px;
}
Methode 2 — Balise style (acceptable)
<style>
  h1 { color: #6B1F2A; }
</style>
Methode 3 — Inline style (a eviter)
<h1 style="color: #6B1F2A;">Titre</h1>
Pourquoi le fichier externe est le mieuxUn seul fichier CSS peut styler des centaines de pages HTML. Si tu veux changer une couleur sur tout le site, tu la changes une seule fois dans le CSS. C'est ca la vraie puissance.

Kora utilise des balises style
Les modules Kora utilisent des balises style dans le head plutot qu'un fichier externe — c'est pour simplifier l'hebergement sur GitHub Pages avec un seul fichier. En production sur un vrai projet, tu utilises toujours un fichier CSS externe.
Une regle CSS c'est un selecteur + des declarations.

Toute la syntaxe CSS suit exactement le meme pattern. Un selecteur qui cible des elements, des accolades, et des declarations propriete:valeur a l'interieur.

CSS — Anatomie d'une regle
h1 {                  /* Selecteur — cible tous les h1 */
  color: red;          /* Propriete : Valeur ; */
  font-size: 32px;
  font-weight: bold;
  margin-bottom: 16px;
}                     /* Accolade fermante */

/* Plusieurs selecteurs, memes styles */
h1, h2, h3 {
  font-family: 'Fraunces', serif;
  line-height: 1.2;
}

Le point-virgule
Le point-virgule a la fin de chaque declaration CSS est obligatoire — sauf pour la derniere declaration d'un bloc (mais mets-le quand meme par habitude). Si tu oublies un point-virgule, les declarations suivantes peuvent etre ignorees. C'est l'erreur numero 1 des debutants.
La cascade — qui gagne quand plusieurs regles s'appliquent ?

CSS signifie Cascading Style Sheets. Le mot "cascading" est crucial — il definit comment les conflits entre regles sont resolus. Trois facteurs : la specificite, l'ordre, l'importance.

CSS — Specificite et cascade
/* Specificite faible — selecteur de balise */
p { color: gray; }

/* Specificite moyenne — classe */
.rouge { color: red; }    /* Gagne sur p */

/* Specificite haute — id */
#titre { color: blue; }  /* Gagne sur .rouge */

/* Inline style — plus fort encore */
/* style="color: green" gagne sur tout */

/* !important — dernier recours, a eviter */
p { color: purple !important; }
L'ordre de specificiteinline style gagne sur #id qui gagne sur .classe qui gagne sur balise. A specificite egale, la regle la plus basse dans le fichier CSS gagne. Mets les regles generales en haut, les exceptions en bas.
🏆
Pourquoi eviter !important
!important force une regle a gagner sur tout. Ca semble pratique mais ca casse la cascade — la logique meme du CSS. Quand tu en abuses, tu te retrouves a mettre !important partout pour surcharger les autres !important. Les bons developpeurs n'utilisent !important que dans des cas tres specifiques.
