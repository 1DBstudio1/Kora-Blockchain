Exercice 🎯
Chaque site que tu utilises — Instagram, Google, Kora — commence par du HTML.

Quand tu ouvres Instagram sur ton téléphone, derrière l'écran il y a du code. Ce code dit au téléphone quoi afficher — les photos, les noms, les boutons. Ce code c'est du HTML à la base.

HTML veut dire HyperText Markup Language. Oublie l'acronyme. Ce qui compte c'est que le HTML c'est un langage qui décrit le contenu d'une page. Il dit "ici c'est un titre", "là c'est une image", "ça c'est un bouton".

Le navigateur — Chrome, Safari, Firefox — lit ce HTML et l'affiche à l'écran. C'est tout le principe.

💡
Le saviez-vous
Tim Berners-Lee a inventé le HTML en 1991 pour que les scientifiques puissent partager des documents entre eux sur internet. Il ne s'attendait pas à ce que ça devienne la base de tout ce qu'on connaît aujourd'hui — Netflix, Amazon, YouTube, tout ça tourne sur du HTML.

Le truc important à comprendre Ces sites ne sont pas "spéciaux". Ils sont construits avec les mêmes outils que toi tu vas apprendre. La différence c'est juste la taille — Instagram a des milliers de fichiers HTML. Toi tu vas commencer avec un seul fichier. Même logique, même langage.

Le HTML c'est juste des instructions dans un fichier texte.

Tu ouvres un éditeur de texte. Tu écris des instructions. Tu sauvegardes en .html. Tu ouvres dans un navigateur. Et là tu vois le résultat.

C'est vraiment aussi simple que ça. Pas de logiciel spécial, pas de serveur compliqué pour commencer. Juste un fichier texte avec les bonnes instructions.

Le navigateur est un traducteur. Il lit ton HTML et le transforme en ce qui s'affiche à l'écran. C'est son seul rôle.

Et quand tu vas sur Google.com, c'est exactement la même chose — mais en sens inverse. Le serveur de Google t'envoie un fichier HTML. Ton navigateur le lit. Et tu vois la page de recherche.

Tout internet fonctionne comme ça. Des fichiers HTML envoyés de serveurs à des navigateurs. À l'échelle, c'est des milliards de fois par seconde. Mais le principe ne change pas.

Essaie maintenant Sur Chrome ou Safari, va sur n'importe quel site — Instagram, YouTube, n'importe quoi. Fais clic droit → "Afficher le code source" ou "Inspecter". Tu verras le HTML de la page. Ce que tu vas apprendre dans cette formation c'est exactement ça que tu vas lire et écrire.
🤯
Chiffre qui claque
Il y a plus de 1,8 milliard de sites web sur internet aujourd'hui. Chacun d'eux a du HTML. Chacun d'eux a commencé par quelqu'un qui a appris exactement ce que tu es en train d'apprendre là.
Le HTML parle avec des balises. C'est là que tout commence.

Une balise c'est une instruction entourée de chevrons < >. Tu dis au navigateur "ce qui est ici c'est quoi".

La majorité des balises fonctionnent en duo — une balise qui ouvre et une balise qui ferme. Le contenu se met entre les deux. La balise fermante a juste un slash / en plus.

HTML — Anatomie d'une balise
<!-- Structure de base d'une balise -->

<p>Ton texte ici</p>
  ↑             ↑
  Ouvre         Ferme (remarque le /)

<!-- D'autres exemples -->

<h1>Titre principal</h1>
<h2>Sous-titre</h2>
<p>Un paragraphe de texte normal.</p>
<strong>Texte en gras</strong>
<em>Texte en italique</em>

Regarde bien — le navigateur a lu tes balises et les a transformées en mise en forme. <h1> est devenu un grand titre en gras. <em> est devenu de l'italique. Tu n'as rien eu à dire "mets ça en gras" — le navigateur sait déjà ce que chaque balise veut dire.

<h1>
Heading 1 — Le titre le plus important de la page. Il ne doit y en avoir qu'un seul par page. Les moteurs de recherche comme Google lui donnent beaucoup d'importance.
<h2> à <h6>
Sous-titres de niveaux décroissants. h2 est moins important que h1, h3 moins que h2, etc. Comme une table des matières.
<p>
Paragraph — Un bloc de texte. Le navigateur ajoute automatiquement de l'espace avant et après. C'est la balise la plus utilisée sur le web.
<strong>
Met le texte en gras et lui donne de l'importance sémantique. Les lecteurs d'écran pour les malvoyants vont accentuer ce mot.
<em>
Met le texte en italique et lui donne de l'emphase. Vient de "emphasis" — tu mets l'accent sur quelque chose.
L'erreur la plus commune Oublier de fermer une balise. Si tu ouvres <p> et tu oublies </p>, le navigateur va essayer de deviner ce que tu voulais faire — et souvent il va faire n'importe quoi. Toujours vérifier que chaque balise ouvrante a sa balise fermante correspondante.
HTML, CSS, JavaScript — trois langages, trois rôles.

Le web est construit sur trois piliers. Chacun a un rôle précis. Les confondre c'est la première erreur des débutants. On va démystifier ça une fois pour toutes.

🏗
HTML
La structure. Ce qui est sur la page — les titres, les textes, les images, les boutons. Le squelette de tout.
Analogie : les murs d'une maison
🎨
CSS
Le style. Comment ça ressemble — les couleurs, les polices, les tailles, les animations. La déco.
Analogie : la peinture et les meubles
⚡
JavaScript
Le comportement. Ce qui se passe quand tu cliques, ce qui bouge, ce qui réagit. L'intelligence de la page.
Analogie : l'électricité et la domotique

Regarde Instagram sur ton téléphone. Quand tu vois la liste de posts — c'est le HTML qui dit "il y a un post ici, une image là, un nom d'utilisateur". Quand tu vois les couleurs, les polices, l'espacement — c'est le CSS. Quand tu scrolles et que de nouveaux posts se chargent automatiquement, ou quand tu double-cliques pour liker — c'est le JavaScript.

Les trois travaillent ensemble sur chaque page que tu visites. Mais ils ont chacun leur fichier, leur rôle, leur logique. Et on les apprend dans l'ordre — HTML d'abord, CSS ensuite, JavaScript après.

HTML — La structure
<!-- HTML : on décrit le contenu -->
<button>Like ❤️</button>
CSS — Le style (tu apprendras ça après)
/* CSS : on décrit comment ça ressemble */
button {
  background: red;
  color: white;
  border-radius: 20px;
}
JavaScript — Le comportement (encore après)
// JavaScript : on décrit ce qui se passe
button.addEventListener('click', () => {
  alert('Tu as liké !');
});
Pourquoi on commence par le HTML Parce que sans structure, le style ne sert à rien. Et sans structure et style, le comportement n'a rien sur quoi s'appuyer. HTML c'est la fondation. Tu ne construis pas un toit avant les murs.

🎯 Ta première page HTML Exercice

Ce que tu dois écrire

  • Un sous-titre <h2> avec la phrase "L'information vérifiée"
  • Un paragraphe <p> qui décrit Kora en une ou deux phrases
  • Un deuxième paragraphe avec le mot "confiance" mis en gras avec <strong>
