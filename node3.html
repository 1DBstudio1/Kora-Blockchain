Express — le framework web Node.js.

Node.js natif peut creer un serveur HTTP mais c'est verbeux. Express simplifie tout : routing, parsing du body, middlewares, erreurs. C'est le framework numero 1 pour les APIs REST en Node.js.

Ce qu'Express apporteSans Express tu dois parser manuellement l'URL, le body, gerer les methodes HTTP, renvoyer les bons headers. Avec Express : app.get('/users', handler) et c'est tout. Toujours dans tous les projets Node.js serieux.
Express — Serveur de base
const express = require('express');
const app = express();
const PORT = process.env.PORT || 3000;

// Parser le JSON des requetes
app.use(express.json());

// Route GET
app.get('/', (req, res) => {
  res.json({ message: 'Kora API v1', status: 'OK' });
});

app.get('/health', (req, res) => {
  res.json({
    uptime: process.uptime(),
    timestamp: new Date().toISOString()
  });
});

app.listen(PORT, () => {
  console.log('Kora API sur http://localhost:' + PORT);
});

// curl http://localhost:3000/
// {"message":"Kora API v1","status":"OK"}
🚀
res.json vs res.send
res.json() serialise en JSON et set Content-Type: application/json. res.send() envoie n'importe quoi. res.status(404).json({error:'Not found'}) pour les erreurs. Toujours terminer par res.json() ou res.send() — sinon la requete reste en attente indefiniment.
Routes REST — les endpoints de ton API.

GET pour lire, POST pour creer, PUT/PATCH pour modifier, DELETE pour supprimer. La convention REST que toutes les APIs modernes suivent.

Convention RESTGET /users — liste. GET /users/:id — un seul. POST /users — creer. PUT /users/:id — remplacer. DELETE /users/:id — supprimer. Ces conventions sont universelles — tout dev qui voit ton API comprend la structure immediatement.
Express — Routes REST
const express = require('express');
const app = express();
app.use(express.json());

let users = [
  { id: '1', nom: 'Valentin', krt: 248 },
];

// GET /users
app.get('/users', (req, res) => res.json(users));

// GET /users/:id
app.get('/users/:id', (req, res) => {
  const user = users.find(u => u.id === req.params.id);
  if (!user) return res.status(404).json({ error: 'Introuvable' });
  res.json(user);
});

// POST /users
app.post('/users', (req, res) => {
  const { nom } = req.body;
  if (!nom) return res.status(400).json({ error: 'nom requis' });
  const u = { id: Date.now().toString(), nom, krt: 0 };
  users.push(u);
  res.status(201).json(u);
});

// DELETE /users/:id
app.delete('/users/:id', (req, res) => {
  users = users.filter(u => u.id !== req.params.id);
  res.json({ success: true });
});

app.listen(3000);

Postman et Thunder Client
Pour tester une API sans front-end : Postman ou Thunder Client (extension VS Code). Tu envoies des requetes GET/POST/PUT/DELETE avec headers et body JSON. Indispensable pour developper une API. Teste : GET http://localhost:3000/users.
req.params, query, body — lire les donnees.

Une requete HTTP transmet des donnees de trois facons : dans l'URL (:id), dans les query params (?page=2), dans le body (JSON). Express les expose proprement.

Valider les donneesNe jamais faire confiance aux donnees du client. Toujours valider : req.body.email existe ? Est-ce une string ? Est-ce un email valide ? Des librairies comme Joi ou Zod simplifient la validation. Sans validation, un user peut envoyer n'importe quoi et potentiellement faire planter le serveur.
Express — req params query body
// Query params : /cryptos?sort=prix&limit=10
app.get('/cryptos', (req, res) => {
  const { sort = 'ticker', limit = 20 } = req.query;
  const n = Math.min(parseInt(limit) || 20, 100);
  res.json({ data: cryptos.slice(0, n) });
});

// Params URL : /users/123/portfolio
app.get('/users/:userId/portfolio', (req, res) => {
  const { userId } = req.params;
  res.json({ userId, positions: [] });
});

// Body JSON (POST)
app.post('/positions', (req, res) => {
  const { ticker, entree, quantite } = req.body;

  if (!ticker || typeof ticker !== 'string') {
    return res.status(400).json({ error: 'ticker invalide' });
  }
  if (!entree || entree <= 0) {
    return res.status(400).json({ error: 'entree invalide' });
  }

  const position = { id: Date.now().toString(), ticker, entree, quantite };
  res.status(201).json(position);
});
🛡️
Status codes
200 OK succes. 201 Created apres POST. 400 Bad Request donnees invalides. 401 Unauthorized pas connecte. 403 Forbidden pas les droits. 404 Not Found ressource inexistante. 500 Internal Server Error erreur non geree. Des bons status codes = API professionnelle.
