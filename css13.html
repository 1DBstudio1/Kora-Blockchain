transition — animer les changements d'etat.

Transition anime le passage d'un style a un autre. Sans transition, les changements au :hover sont instantanes. Avec transition, l'element glisse progressivement vers le nouveau style — c'est ce qui donne une impression de qualite et de fluidite.

Survole cette card — elle a transition: transform 0.3s ease, box-shadow 0.3s ease
CSS — transition
/* Syntaxe : propriete duree timing-function delai */
.btn {
  background: #6B1F2A;
  transition: all 0.2s ease;
  /* 'all' anime toutes les proprietes qui changent */
}
.btn:hover {
  background: #9B2F3A;
  transform: translateY(-2px);
}

/* Meilleure pratique : transitions specifiques */
.card {
  transition:
    transform 0.3s ease,
    box-shadow 0.3s ease,
    border-color 0.2s ease;
}
.card:hover {
  transform: translateY(-4px);
  box-shadow: 0 16px 48px rgba(0,0,0,0.1);
  border-color: rgba(196,98,45,0.3);
}

/* Les timing functions */
/* ease : lent-rapide-lent (defaut, le plus naturel) */
/* linear : vitesse constante */
/* ease-in : demarre lentement */
/* ease-out : finit lentement */
/* ease-in-out : lent au debut et a la fin */
Transition specifique vs alltransition: all est pratique mais anime TOUTES les proprietes qui changent — meme celles qui ne devraient pas. Prefere lister les proprietes specifiques : transition: transform 0.2s ease, box-shadow 0.2s ease. C'est aussi plus performant car le navigateur sait exactement quoi animer.
transform — deplacer, agrandir, tourner sans bouger le flux.

transform modifie geometriquement un element sans affecter le flux de la page. Contrairement a changer top ou margin, transform ne force pas le navigateur a recalculer tout le layout.

CSS — transform
/* Deplacement */
.el { transform: translateX(20px); }    /* Droite */
.el { transform: translateY(-10px); }   /* Haut */
.el { transform: translate(-50%, -50%); } /* Centrage */

/* Agrandissement */
.btn:hover   { transform: scale(1.05); }  /* +5% */
.btn:active  { transform: scale(0.95); }  /* Effet clic */

/* Rotation */
.icone       { transform: rotate(45deg); }
.icone.open  { transform: rotate(180deg); } /* Chevron */

/* Combiner plusieurs transforms */
.card:hover {
  transform: translateY(-4px) scale(1.02);
}

/* transform-origin — point de pivot */
.el { transform-origin: top left; } /* Tourne depuis coin haut-gauche */

Performance : transform et opacity
transform et opacity sont les deux proprietes CSS les plus performantes pour les animations. Le navigateur peut les animer directement sur le GPU, sans recalculer le layout de la page. Evite d'animer width, height, top, left, margin, padding — ca force un "layout reflow" complet a chaque frame et cause des saccades. La regle : anime seulement transform et opacity pour des animations a 60fps.
@keyframes — des animations qui se jouent toutes seules.

@keyframes definit les etapes d'une animation. La propriete animation applique cette animation a un element. Contrairement a transition, les animations @keyframes ne necessitent pas d'interaction utilisateur — elles jouent automatiquement.

CSS — @keyframes et animation
/* 1. Definir l'animation */
@keyframes slideUp {
  from {
    opacity: 0;
    transform: translateY(20px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}

/* 2. Appliquer l'animation */
.carte {
  animation: slideUp 0.4s ease both;
  /* nom  duree  timing  fill-mode */
}

/* Animation avec etapes en % */
@keyframes pulse {
  0%, 100% { transform: scale(1); opacity: 1; }
  50%       { transform: scale(1.06); opacity: 0.8; }
}
.badge-krt {
  animation: pulse 2s ease infinite; /* Boucle infinie */
}

/* Skeleton loading */
@keyframes shimmer {
  0%   { background-position: -200% 0; }
  100% { background-position: 200% 0; }
}
.skeleton {
  background: linear-gradient(90deg, #f0f0f0 25%, #e0e0e0 50%, #f0f0f0 75%);
  background-size: 200% 100%;
  animation: shimmer 1.5s infinite;
}

Animations avancees — controle total avec animation.

La propriete animation est un raccourci pour plusieurs sous-proprietes. Tu peux controler la direction, le mode de remplissage, le delai, et meme enchainner plusieurs animations.

CSS — animation proprietes detaillees
/* Raccourci complet */
.el {
  animation: slideUp 0.4s ease 0.1s both;
  /* nom duree timing delai fill-mode */
}

/* Proprietes separees */
.el {
  animation-name: slideUp;
  animation-duration: 0.4s;
  animation-timing-function: ease;
  animation-delay: 0.1s;          /* Delai avant de commencer */
  animation-fill-mode: both;       /* Garde style initial et final */
  animation-iteration-count: 1;    /* Ou 'infinite' */
  animation-direction: normal;     /* Ou 'reverse', 'alternate' */
  animation-play-state: running;   /* Ou 'paused' pour mettre en pause */
}

/* Plusieurs animations */
.el {
  animation: slideUp 0.4s ease, pulse 2s ease 0.5s infinite;
}

/* ACCESSIBILITE — respecter les preferences */
@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
animation-fill-mode: bothboth est presque toujours ce que tu veux. "both" = l'element garde le style du premier keyframe avant de commencer (forward), et garde le style du dernier keyframe apres la fin (backwards). Sans ca, l'element revient a son etat initial apres l'animation — ce qui cause un saut visuel desagreable.
♿
prefers-reduced-motion
Certains utilisateurs (epileptiques, sensibles aux mouvements, migraineux) peuvent activer "reduire les animations" dans leurs parametres d'accessibilite. La media query prefers-reduced-motion: reduce detecte ce parametre. Bonne pratique : ajoute cette media query dans tous tes CSS pour desactiver ou ralentir les animations pour ces utilisateurs. C'est un geste simple qui peut changer leur experience.

  • transition sur les boutons et liens au :hover (transform + box-shadow)
  • transform: translateY(-Npx) sur les cards au :hover avec transition
  • Un @keyframes d'entree (fadeIn ou slideUp)
  • animation: appliquee sur au moins un element
