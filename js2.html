let, const et var — declarer des variables.

En JavaScript moderne tu declares des variables avec let ou const. let pour une valeur qui peut changer. const pour une valeur fixe. var est l'ancienne syntaxe — evite-la, elle a des comportements surprenants.

JavaScript — let et const
// const — valeur constante (ne peut pas changer)
const nom = 'Valentin';
const prixBTC = 65000;
const PI = 3.14159;

// let — valeur variable (peut changer)
let score = 0;
score = 42;          // OK — let peut etre reassigne

let estConnecte = false;
estConnecte = true; // OK

// const ne peut pas etre reassigne
// prixBTC = 70000; → TypeError !

// Regles de nommage
const maVariable = 1;  // camelCase — convention JS
const MA_CONSTANTE = 1; // UPPER_SNAKE pour constantes globales
// Pas d'espaces, pas de tirets, pas commencer par un chiffre
const par defaut, let si necessaireLa bonne pratique : utilise toujours const en premier. Si tu te rends compte que la valeur doit changer, passe en let. Ca force a reflechir a la mutabilite des donnees et reduit les bugs. En pratique, environ 80% des variables d'un code propre sont des const.
⚠️
Pourquoi eviter var
var a deux problemes majeurs. 1) Le hoisting : var est "hissee" en haut de la fonction, ce qui cree des comportements contre-intuitifs. 2) La portee : var ignore les blocs if et for — elle est accessible partout dans la fonction. let et const ont une portee de bloc, ce qui est plus logique. Depuis ES6 (2015), il n'y a aucune raison d'utiliser var dans du code moderne.
JavaScript a 7 types de donnees fondamentaux.

Chaque valeur en JavaScript a un type. Le type determine ce qu'on peut faire avec la valeur. Connaitre les types c'est indispensable pour comprendre pourquoi certaines operations marchent et d'autres pas.

string
Chaine de caracteres
Du texte entre guillemets simples, doubles, ou backticks.
'Kora' "Bitcoin" `prix: ${p}`
number
Nombre
Entiers et decimaux. Un seul type pour les deux.
42 3.14 -7 1_000_000
boolean
Booleen
true ou false. La base de toute logique conditionnelle.
true false
null
Null
Absence intentionnelle de valeur. "Pas de valeur, intentionnellement."
null
undefined
Undefined
Variable declaree mais sans valeur assignee.
let x; // x est undefined
object
Objet
Collection de paires cle:valeur. Les tableaux sont aussi des objets.
{nom: 'BTC', prix: 65000}
JavaScript — typeof
// typeof retourne le type d'une valeur
typeof 'Kora'      // → 'string'
typeof 65000      // → 'number'
typeof true       // → 'boolean'
typeof null       // → 'object' (bug historique de JS !)
typeof undefined  // → 'undefined'
typeof {}         // → 'object'
typeof []         // → 'object'
typeof function(){} // → 'function'

console.log(typeof '42' === 'string'); // → true
console.log(typeof 42 === 'number');   // → true
typeof null === 'object'C'est un bug historique de JavaScript qui n'a jamais ete corrige pour ne pas casser le code existant. Pour verifier si quelque chose est null, utilise valeur === null plutot que typeof valeur === 'null'.
Les strings — manipuler le texte en JavaScript.

Les strings sont partout — noms d'utilisateur, messages, donnees venant d'APIs, contenu HTML. JavaScript a des dizaines de methodes pour les manipuler. Les plus utiles sont length, toUpperCase, toLowerCase, includes, slice, split, trim, replace.

JavaScript — Methodes de string
const crypto = 'Bitcoin';

// Longueur
crypto.length           // → 7

// Casse
crypto.toUpperCase()    // → 'BITCOIN'
crypto.toLowerCase()    // → 'bitcoin'

// Chercher
crypto.includes('coin') // → true
crypto.startsWith('Bit')// → true
crypto.indexOf('t')     // → 3 (position du 't')

// Extraire
crypto.slice(0, 3)      // → 'Bit' (de 0 a 3 exclu)
crypto.slice(-4)        // → 'coin' (4 depuis la fin)

// Remplacer
crypto.replace('Bit', 'Alt') // → 'Altcoin'

// Supprimer espaces
'  kora  '.trim()       // → 'kora'

// Diviser en tableau
'BTC,ETH,SOL'.split(',') // → ['BTC', 'ETH', 'SOL']

Les strings sont immutables
Quand tu appelles crypto.toUpperCase(), tu n'as pas modifie la variable crypto — tu as cree une nouvelle string. Les strings JavaScript sont immutables : leurs methodes retournent toujours de nouvelles valeurs sans modifier l'original. Si tu veux la version uppercase, tu dois la stocker : const cryptoMaj = crypto.toUpperCase().
Template literals — construire des strings dynamiques.

Les template literals (backticks `) permettent d'inserer des variables directement dans une string avec ${variable}. Plus lisible que la concatenation avec +, et supportent les strings multiligne nativement.

JavaScript — Template literals
const prenom = 'Valentin';
const krt = 248;
const btc = 65000;

// Concatenation classique (ancien style)
console.log('Bonjour ' + prenom + ', tu as ' + krt + ' KRT');

// Template literal (moderne, recommande)
console.log(`Bonjour ${prenom}, tu as ${krt} KRT`);

// Expressions dans ${}
console.log(`BTC : ${btc.toLocaleString('fr-FR')} €`);
console.log(`Double KRT : ${krt * 2}`);

// Multilignes
const message = `
  Bienvenue ${prenom} !
  Solde : ${krt} KRT
  BTC : ${btc} $
`;

// Construire du HTML dynamiquement
const html = `<div class="card">
  <h3>${prenom}</h3>
  <p>${krt} KRT</p>
</div>`;
