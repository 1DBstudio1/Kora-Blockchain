Les media queries — du CSS conditionnel selon l'ecran.

Une media query dit : "applique ce CSS SEULEMENT si cette condition est vraie". La condition principale c'est la largeur de l'ecran. Tu peux aussi cibler l'orientation, la resolution, ou meme la preference de mode sombre.

Syntaxe : @media (condition) suivi d'un bloc CSS qui ne s'applique que si la condition est remplie.

CSS — Media queries
/* Styles de base — s'appliquent partout */
.titre { font-size: 24px; }

/* Si l'ecran fait AU MOINS 768px (tablette+) */
@media (min-width: 768px) {
  .titre { font-size: 36px; }
}

/* Si l'ecran fait AU PLUS 768px (mobile) */
@media (max-width: 768px) {
  .menu { display: none; }
}

/* Plusieurs conditions */
@media (min-width: 768px) and (max-width: 1024px) {
  /* Seulement tablette */
}

/* Orientation */
@media (orientation: landscape) {
  /* Ecran en mode paysage */
}

/* Mode sombre systeme */
@media (prefers-color-scheme: dark) {
  body { background: #0A0A0A; }
}
La balise meta viewport — indispensableSans cette balise dans le head HTML, les telephones affichent la page a 980px de large et zooment out. Avec elle, la page s'affiche a la vraie largeur de l'ecran. C'est la condition sine qua non du responsive. Tu l'as deja dans tous tes fichiers Kora : <meta name="viewport" content="width=device-width, initial-scale=1.0">

60% mobile en 2024
En 2024, 60 a 70% du trafic web mondial vient des smartphones. Sur certains sites (actualites, reseaux sociaux, e-commerce), c'est plus de 80%. Si ton site n'est pas responsive, tu perds la majorite de tes visiteurs. Google indexe d'abord la version mobile des sites depuis 2020 — "mobile first indexing". Un site non-responsive est penalise dans les resultats de recherche.
Mobile first — partir du petit et aller vers le grand.

Il y a deux approches : desktop first (tu ecris pour grand ecran, puis tu adaptes vers le bas) et mobile first (tu ecris pour mobile, puis tu ameliores vers le grand). Mobile first est l'approche recommandee — le CSS mobile est plus simple, et tu ajoutes de la complexite au fur et a mesure.

CSS — Mobile first en pratique
/* APPROCHE MOBILE FIRST */

/* Base = mobile (pas de media query) */
.cards {
  display: grid;
  grid-template-columns: 1fr; /* 1 colonne sur mobile */
  gap: 16px;
}
.nav-links { display: none; } /* Cache sur mobile */

/* Tablette (640px et plus) */
@media (min-width: 640px) {
  .cards { grid-template-columns: repeat(2, 1fr); }
}

/* Desktop (1024px et plus) */
@media (min-width: 1024px) {
  .cards { grid-template-columns: repeat(3, 1fr); }
  .nav-links { display: flex; } /* Visible sur desktop */
}

min-width vs max-widthMobile first = utilise min-width dans tes media queries ("a partir de X, applique ca"). Desktop first = utilise max-width ("jusqu'a X, applique ca"). Mobile first est recommande car le CSS de base (sans media query) est plus simple — une colonne, pas de sidebar, texte plus grand. Tu complexifies vers le haut.
Les unites relatives — s'adapter automatiquement.

Les unites relatives s'adaptent a leur contexte. px est fixe — 16px reste 16px sur tous les ecrans. rem est relatif a la taille de police du html. vw et vh sont relatifs a la taille du viewport. % est relatif au parent.

px
Pixels — fixe
16px reste 16px partout. Utilise pour les bordures, les petits elements fixes. A eviter pour les polices et les espacements principaux.
rem
Root em — relatif au html
1rem = taille de police du html (defaut 16px). 2rem = 32px. Parfait pour les polices et espacements — respecte les preferences utilisateur.
%
Pourcentage — relatif au parent
50% = moitie du conteneur parent. Parfait pour les largeurs. width:100% prend toute la largeur disponible.
vw/vh
Viewport — relatif a l'ecran
1vw = 1% de la largeur. 1vh = 1% de la hauteur. height:100vh = toute la hauteur de l'ecran.
em
Em — relatif au parent
1em = taille de police de l'element parent. Peut devenir impredictible dans les elements imbriques. Prefere rem.
clamp()
Clamp — responsive sans query
clamp(min, preferred, max). font-size: clamp(16px, 2.5vw, 24px). Jamais moins de 16px, jamais plus de 24px. Responsive automatique.
CSS — Unites relatives en pratique
html { font-size: 16px; } /* Base pour rem */

body {
  font-size: 1rem;    /* = 16px */
  line-height: 1.6;
}

h1 {
  /* Responsive sans media query */
  font-size: clamp(24px, 5vw, 56px);
}

.hero {
  height: 100vh;   /* Plein ecran */
  padding: 0 5%;  /* 5% des cotes */
}

.container {
  max-width: 1200px;
  width: 100%;
  margin: 0 auto;
  padding: 0 1.5rem;
}

Tailwind CSS et les breakpoints
Tailwind CSS, le framework CSS le plus populaire en 2024, utilise exactement les memes breakpoints que ce qu'on voit dans ce module — sm: (640px), md: (768px), lg: (1024px), xl: (1280px), 2xl: (1536px). Maintenant que tu comprends les media queries, tu comprends toute la logique derriere Tailwind. C'est juste des utilitaires CSS pre-construits sur ce meme systeme.
Les breakpoints — les largeurs d'ecran importantes.

Un breakpoint c'est une largeur d'ecran a partir de laquelle le design change. Il n'y a pas de valeurs universelles — ca depend de ton design. Mais les breakpoints suivants correspondent aux appareils reels et sont les plus utilises dans l'industrie.

AppareilBreakpointUtilisation
📱640px
Tablette portrait. Passer de 1 a 2 colonnes, agrandir les polices.
📱768px
Tablette paysage. Afficher la navigation horizontale, sidebar apparait.
💻1024px
Laptop. Passer a 3 colonnes, layout complet avec sidebar.
🖥1280px
Desktop standard. Taille max du conteneur, espacements plus genereux.
🖥1536px
Grand ecran. 4 colonnes, max-width plus large.
CSS — Layout responsive complet
/* Mobile first — base sans media query */
.page {
  padding: 16px;
}
.cards {
  display: grid;
  grid-template-columns: 1fr;
  gap: 12px;
}
.nav-links { display: none; }

/* Tablette */
@media (min-width: 768px) {
  .page { padding: 24px; }
  .cards { grid-template-columns: repeat(2, 1fr); }
  .nav-links { display: flex; }
}

/* Desktop */
@media (min-width: 1024px) {
  .page { max-width: 1200px; margin: 0 auto; padding: 40px; }
  .cards { grid-template-columns: repeat(3, 1fr); gap: 20px; }
}
🎯Rends ta page profil responsiveExercice
Ecris le CSS responsive pour ta page profil. Elle doit fonctionner sur mobile (1 colonne, texte plus petit) et sur desktop (layout plus large, typographie plus grande). Utilise l'approche mobile first.
Ton CSS doit avoir
  • Styles de base pour mobile sans media query (padding, font-size, 1 colonne)
  • Au moins une media query avec min-width pour tablette (640px ou 768px)
  • Au moins une media query avec min-width pour desktop (1024px)
  • Une unite relative — rem, vw, %, ou clamp() — sur au moins une valeur
  • Le layout qui change entre mobile et desktop (colonnes, tailles...)
