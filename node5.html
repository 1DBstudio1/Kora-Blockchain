SQL — le langage universel des BDD.

SQL est utilise par SQLite, PostgreSQL, MySQL. Apprendre SQL c'est apprendre a parler a n'importe quelle BDD relationnelle. Les concepts sont les memes partout.

Relationnel vs NoSQLBDD relationnelles (PostgreSQL, SQLite) = tables avec relations. NoSQL (MongoDB, Redis) = documents ou cle-valeur. Pour Kora : PostgreSQL pour les donnees structurees. Redis pour le cache. La majorite des apps web utilisent une BDD relationnelle en coeur.

SQLite vs PostgreSQL
SQLite = un seul fichier .db. Zero serveur, zero config. Parfait pour dev. Limite : pas adapte a des milliers de requetes simultanees. PostgreSQL = vrai serveur, scalable, features avancees. Pour Kora prod : PostgreSQL. Pour apprendre : SQLite. Meme SQL, migration facile avec Prisma.
better-sqlite3 — SQLite dans Node.js.

La librairie better-sqlite3 utilise SQLite depuis Node.js. Version synchrone, la plus simple pour commencer. En production on passera a PostgreSQL.

Prepared statements — securiteNe jamais concatener les donnees utilisateur dans une requete SQL directement. C'est une injection SQL. Toujours utiliser des prepared statements avec ? : db.prepare('SELECT * FROM users WHERE email = ?').get(email). La librairie echappe automatiquement.
Node.js + SQLite
// npm install better-sqlite3

const Database = require('better-sqlite3');
const db = new Database('kora.db');

// Creer les tables au demarrage
db.exec(`
  CREATE TABLE IF NOT EXISTS users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    nom TEXT NOT NULL,
    email TEXT UNIQUE NOT NULL,
    password_hash TEXT NOT NULL,
    krt INTEGER DEFAULT 0,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
  )
`);

// Prepared statements
const insertUser = db.prepare(
  'INSERT INTO users (nom, email, password_hash) VALUES (?, ?, ?)'
);
const getUserByEmail = db.prepare(
  'SELECT * FROM users WHERE email = ?'
);
const addKrt = db.prepare(
  'UPDATE users SET krt = krt + ? WHERE id = ?'
);

// Utilisation
function creerUser(nom, email, hash) {
  const r = insertUser.run(nom, email, hash);
  return { id: r.lastInsertRowid, nom, email, krt: 0 };
}

function trouver(email) {
  return getUserByEmail.get(email);
}

Transactions SQL
Une transaction groupe plusieurs requetes — toutes reussissent ou toutes echouent. Essentiel pour les operations critiques comme les transferts de KRT. db.transaction(() => { deduire(from); ajouter(to); })(). Si une requete plante, tout est annule.
Prisma — l'ORM moderne.

Prisma traduit tes objets JavaScript en SQL. Schema declaration, migrations, client type-safe. En production avec PostgreSQL c'est ce qu'on utilisera.

Schema PrismaTu definis le schema dans schema.prisma. Prisma genere le client JS et les migrations SQL. Si tu changes le schema : npx prisma migrate dev. npx prisma studio ouvre une interface graphique pour voir les donnees. Workflow moderne — plus besoin d'ecrire le SQL a la main.
Prisma — Schema et queries
// schema.prisma
generator client { provider = "prisma-client-js" }
datasource db {
  provider = "sqlite"
  url      = env("DATABASE_URL")
}

model User {
  id        Int      @id @default(autoincrement())
  nom       String
  email     String   @unique
  krt       Int      @default(0)
  createdAt DateTime @default(now())
  progressions Progression[]
}

model Progression {
  id       Int    @id @default(autoincrement())
  userId   Int
  moduleId String
  statut   String @default("in_progress")
  score    Int?
  user     User   @relation(fields:[userId], references:[id])
}

// Queries Prisma
const { PrismaClient } = require('@prisma/client');
const prisma = new PrismaClient();

const user = await prisma.user.create({
  data: { nom: 'Valentin', email: 'val@kora.fr' }
});

await prisma.user.update({
  where: { id: 1 },
  data: { krt: { increment: 5 } }
});
⚡
DATABASE_URL en .env
Ne jamais hardcoder la connexion BDD. Fichier .env : DATABASE_URL=file:./kora.db. Ajoute .env dans .gitignore. npm install dotenv, require('dotenv').config() au debut de index.js. En production les variables sont configurees dans l'interface du hosting.
