Les tableaux — des listes ordonnees.

Un tableau (array) stocke plusieurs valeurs dans un ordre precis. Chaque element a un index qui commence a 0. Les tableaux sont la structure de donnees la plus utilisee en JavaScript.

JavaScript — Tableaux
// Creer un tableau
const cryptos = ['BTC', 'ETH', 'SOL', 'XRP'];
const prix = [65000, 3200, 170, 0.58];
const mixte = ['BTC', 65000, true, null];

// Acceder par index (debut a 0)
cryptos[0]         // → 'BTC'
cryptos[2]         // → 'SOL'
cryptos[cryptos.length - 1] // → 'XRP' (dernier)
cryptos.at(-1)     // → 'XRP' (moderne)

// Modifier un element
cryptos[1] = 'MATIC';

// Longueur
cryptos.length      // → 4

// Ajouter / supprimer
cryptos.push('BNB');    // Ajouter a la fin
cryptos.pop();          // Supprimer le dernier
cryptos.unshift('BTC');// Ajouter au debut
cryptos.shift();        // Supprimer le premier

JSON — le format des APIs
Quand tu appelles une API crypto (CoinGecko, Binance...), la reponse est du JSON. JSON c'est des objets et des tableaux JavaScript serialises en texte. Un tableau JSON c'est exactement comme un tableau JS. Comprendre les tableaux et objets JS c'est comprendre comment lire et manipuler toutes les donnees qui viennent d'APIs — prix crypto, donnees de marche, profils utilisateurs.
map, filter, find — les trois methodes qui changent tout.

Ces trois methodes fonctionnelles transforment les tableaux sans les modifier. Elles prennent une fonction en argument (callback) et retournent un nouveau tableau. Elles sont au coeur de la programmation JS moderne.

JavaScript — map, filter, find, reduce
const cryptos = [
  { ticker: 'BTC', prix: 65000, variation: 2.3 },
  { ticker: 'ETH', prix: 3200,  variation: -1.1 },
  { ticker: 'SOL', prix: 170,   variation: 5.7 },
  { ticker: 'XRP', prix: 0.58,  variation: -0.4 },
];

// map — transformer chaque element
const tickers = cryptos.map(c => c.ticker);
// → ['BTC', 'ETH', 'SOL', 'XRP']

// filter — garder les elements qui passent le test
const enHausse = cryptos.filter(c => c.variation > 0);
// → [BTC, SOL] (seulement les positifs)

// find — le premier qui passe le test
const btc = cryptos.find(c => c.ticker === 'BTC');
// → {ticker: 'BTC', prix: 65000, variation: 2.3}

// reduce — aggreger en une valeur
const totalPortfolio = cryptos.reduce((acc, c) => acc + c.prix, 0);
// → 68370.58

// sort — trier
const trie = [...cryptos].sort((a, b) => b.prix - a.prix);
// → Trie par prix decroissant

Les objets — des paires cle:valeur.

Un objet regroupe des proprietes liees. Une crypto a un ticker, un prix, une variation. Un utilisateur a un nom, un email, un score. Les objets modelisent les entites du monde reel.

JavaScript — Objets
// Creer un objet
const btc = {
  ticker: 'BTC',
  nom: 'Bitcoin',
  prix: 65000,
  variation: 2.3,
  enHausse: true,
};

// Acceder aux proprietes
btc.ticker         // → 'BTC' (notation pointee)
btc['prix']       // → 65000 (notation crochet)

// Modifier
btc.prix = 70000;

// Ajouter une propriete
btc.marketCap = 1.28e12;

// Verifier si une cle existe
'ticker' in btc    // → true

// Lister les cles, valeurs, entrees
Object.keys(btc)    // → ['ticker','nom','prix',...]
Object.values(btc)  // → ['BTC','Bitcoin',65000,...]
Object.entries(btc) // → [['ticker','BTC'],...]

// Methodes dans un objet
const crypto = {
  prix: 65000,
  quantite: 0.5,
  valeur() { return this.prix * this.quantite; }
};
crypto.valeur(); // → 32500
Notation pointee vs crochetobj.propriete est la notation habituelle. obj['propriete'] est utile quand le nom de la cle est dans une variable : obj[maVariable]. Ou quand la cle a des espaces ou tirets (a eviter en pratique). Les deux donnent le meme resultat.
Destructuring et spread — la syntaxe moderne indispensable.

Le destructuring extrait des valeurs d'un objet ou tableau en une ligne. Le spread operator (...) copie ou merge des objets et tableaux. Ces syntaxes modernes rendent le code beaucoup plus concis.

JavaScript — Destructuring et spread
// Destructuring objet
const btc = { ticker: 'BTC', prix: 65000, variation: 2.3 };

// Avant (verbeux)
const ticker = btc.ticker;
const prix = btc.prix;

// Apres (destructuring)
const { ticker, prix, variation } = btc;
// Avec valeur par defaut
const { ticker, volume = 0 } = btc;

// Destructuring tableau
const [premier, deuxieme, ...reste] = ['BTC', 'ETH', 'SOL', 'XRP'];
// premier = 'BTC', deuxieme = 'ETH', reste = ['SOL', 'XRP']

// Spread — copier et merger des objets
const base = { ticker: 'BTC', prix: 65000 };
const complet = { ...base, variation: 2.3, enHausse: true };

// Mettre a jour sans muter
const miseAJour = { ...base, prix: 70000 };
// base.prix est toujours 65000

// Merger des tableaux
const t1 = ['BTC', 'ETH'];
const t2 = ['SOL', 'XRP'];
const tous = [...t1, ...t2]; // → ['BTC','ETH','SOL','XRP']

Immutabilite en React
React exige qu'on ne mute pas les objets et tableaux directement — on cree toujours de nouvelles copies. Le spread operator est la solution : const newState = { ...state, prix: 70000 } cree un nouvel objet avec le prix mis a jour. C'est la raison pour laquelle destructuring et spread sont si importants a maitriser avant d'apprendre React ou Vue.
