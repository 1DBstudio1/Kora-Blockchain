C'est quoi Node.js.

JavaScript sans navigateur — c'est ca Node.js.

Node.js utilise le meme moteur V8 que Chrome mais tourne directement sur ton systeme. Pas de DOM, pas de window. A la place : acces aux fichiers, aux sockets reseau, aux processus. C'est la base de tout back-end JavaScript.

Pourquoi c'est important pour KoraLe front-end Kora c'est du HTML/CSS/JS dans le navigateur. Le back-end c'est Node.js sur un serveur — il recoit les requetes, interroge la base de donnees, repond en JSON. Sans Node.js, pas d'auth, pas de profils persistants, pas de tokens KRT reels.
Node.js vs Navigateur
// NAVIGATEUR — ce qu'on a fait
document.querySelector('#titre').textContent = 'Kora';
localStorage.setItem('key', 'val');

// NODE.JS — ce qu'on va apprendre
const fs = require('fs');
fs.writeFileSync('kora.txt', 'Bonjour Kora');

const http = require('http');
http.createServer((req, res) => {
  res.end('Hello from Kora server');
}).listen(3000);

// process — l'objet global Node.js
process.version      // Version Node
process.env.PORT     // Variables d'env
process.argv         // Arguments CLI
process.exit(0)      // Terminer

Node.js partout
Netflix, LinkedIn, Uber, PayPal utilisent Node.js en production. npm est le plus grand registre de packages au monde avec plus de 2 millions de librairies. C'est l'ecosysteme le plus large du developpement web. Apres JS front-end, Node.js est l'etape la plus naturelle pour devenir full-stack.
npm — le gestionnaire de packages.

npm est installe avec Node.js. Il installe des librairies — express, jsonwebtoken, bcrypt, prisma — en une commande. Chaque projet a un package.json qui liste ses dependances.

package.json — le coeur du projetnpm init cree un package.json. node_modules/ contient les librairies — ne jamais le committer. .gitignore doit avoir node_modules/. npm install apres un git clone reinstalle tout depuis package.json.
npm — commandes essentielles
# Initialiser un projet
npm init -y

# Installer Express
npm install express

# Dev uniquement (pas en prod)
npm install --save-dev nodemon

# package.json
{
  "name": "kora-backend",
  "version": "1.0.0",
  "dependencies": {
    "express": "^4.18.2"
  },
  "devDependencies": {
    "nodemon": "^3.0.0"
  },
  "scripts": {
    "start": "node index.js",
    "dev": "nodemon index.js"
  }
}
📦
nodemon
nodemon surveille tes fichiers et redemarre le serveur a chaque modification. Sans ca tu fais Ctrl+C + node index.js apres chaque changement. npm run dev avec nodemon = ta boucle de dev. En production : pm2 qui garde le serveur en vie meme s'il plante.
Premier script — hello world Node.js.

Tu crees un fichier .js, tu l'executes avec node fichier.js dans le terminal. Pas de navigateur, pas de HTML. Juste du JS qui tourne sur ta machine.

require vs importNode.js historiquement utilise require() (CommonJS). Les projets modernes utilisent import/export (ES Modules). Ajoute type: module dans package.json pour les ES Modules. Express supporte les deux. Dans ce cours on utilise require() — plus simple pour commencer, compatible avec toutes les librairies.
Node.js — Premier script
// index.js

const os = require('os');
const path = require('path');

// Infos systeme
console.log('Node:', process.version);
console.log('OS:', os.platform());
console.log('RAM libre:', Math.round(os.freemem()/1024/1024)+'MB');

// Arguments CLI
// node index.js arg1 arg2
const args = process.argv.slice(2);
console.log('Args:', args);

// PORT depuis l'environnement (ou 3000 par defaut)
const PORT = process.env.PORT || 3000;
console.log('Port:', PORT);

// Executer : node index.js
🔧
process — l'objet global
process.env contient les variables d'environnement (mots de passe, cles API — jamais hardcodes). process.argv les arguments CLI. process.exit(0) termine le programme. process.cwd() le dossier courant. C'est le premier objet a connaitre en Node.js.
