La blockchain Ethereum — un ordinateur mondial partagé.

Ethereum = reseau de milliers de noeuds qui executent le meme code de facon decentralisee. Les smart contracts sont des programmes deployes sur ce reseau — immuables et transparents. Le token KRT sera un ERC-20 — le standard Ethereum.

Testnet avant mainnetNe jamais deployer directement sur mainnet — ca coute de vrais ETH. Utilise d'abord Sepolia ou Goerli (testnets gratuits). Outils : Hardhat (dev), MetaMask (wallet), Alchemy ou Infura (acces reseau).
Solidity — Token KRT ERC-20
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.19;

contract KoraToken {
    string public name = "Kora Token";
    string public symbol = "KRT";
    uint8 public decimals = 18;
    uint256 public totalSupply;

    mapping(address => uint256) public balanceOf;
    address public owner;

    event Transfer(address indexed from, address indexed to, uint256 value);
    event Mint(address indexed to, uint256 value);

    constructor() { owner = msg.sender; }

    modifier onlyOwner() {
        require(msg.sender == owner, "Not owner");
        _;
    }

    // Appele par le serveur Kora apres certification
    function mint(address to, uint256 amount) external onlyOwner {
        totalSupply += amount;
        balanceOf[to] += amount;
        emit Mint(to, amount);
        emit Transfer(address(0), to, amount);
    }

    function transfer(address to, uint256 amount) external returns (bool) {
        require(balanceOf[msg.sender] >= amount, "Insufficient");
        balanceOf[msg.sender] -= amount;
        balanceOf[to] += amount;
        emit Transfer(msg.sender, to, amount);
        return true;
    }
}

Gas — le carburant
Chaque operation coute du gas (en ETH). Un mint ERC-20 ~ 50 000 gas. Solution pour Kora : Polygon (meme Solidity, fees 100x moins chers) ou Base (L2 d'Ethereum). Les testnets sont gratuits.
Hardhat — framework de dev Solidity.

Hardhat compile, teste et deploie les smart contracts. Il simule une blockchain locale. Tests en JavaScript.

ABI — l'interface du contractLa compilation genere un ABI — JSON decrivant les fonctions du contract. Node.js utilise l'ABI pour appeler les fonctions. ABI + adresse du contract deploye = interaction depuis n'importe quelle app.

// test/KoraToken.test.js
const { expect } = require('chai');
const { ethers } = require('hardhat');

describe('KoraToken', () => {
  let token, owner, addr1;

  beforeEach(async () => {
    [owner, addr1] = await ethers.getSigners();
    const KRT = await ethers.getContractFactory('KoraToken');
    token = await KRT.deploy();
  });

  it('mint des tokens', async () => {
    await token.mint(addr1.address, ethers.parseEther('5'));
    expect(await token.balanceOf(addr1.address))
      .to.equal(ethers.parseEther('5'));
  });
});

// scripts/deploy.js
async function main() {
  const KRT = await ethers.getContractFactory('KoraToken');
  const token = await KRT.deploy();
  await token.waitForDeployment();
  console.log('KRT deploye:', await token.getAddress());
}
main();

Wallet et cle privee
Deployer necessite un wallet avec des ETH (ou MATIC). La cle privee signe les transactions. Jamais dans le code — dans .env : PRIVATE_KEY=0x... Le wallet qui deploie devient l'owner — il peut minter les KRT.
ethers.js — interagir avec la blockchain depuis Node.js.

ethers.js permet d'appeler les fonctions du smart contract depuis Node.js. Quand un user est certifie, le serveur appelle mint() pour crediter ses KRT reels.

On-chain vs off-chainApproche hybride : donnees de profil/progression dans PostgreSQL (off-chain, rapide). Tokens KRT et certifications officielles sur la blockchain (on-chain, verifiable). Le serveur synchronise les deux.
Node.js + ethers.js
// npm install ethers

const { ethers } = require('ethers');

const KRT_ABI = [
  'function mint(address to, uint256 amount) external',
  'function balanceOf(address account) view returns (uint256)',
  'event Mint(address indexed to, uint256 value)'
];

const provider = new ethers.JsonRpcProvider(process.env.RPC_URL);
const wallet = new ethers.Wallet(process.env.PRIVATE_KEY, provider);
const krt = new ethers.Contract(process.env.KRT_ADDRESS, KRT_ABI, wallet);

// Minter 5 KRT apres certification
async function minterKRT(walletAddress, montant = 5) {
  try {
    const amount = ethers.parseEther(montant.toString());
    const tx = await krt.mint(walletAddress, amount);
    await tx.wait(); // Attendre la confirmation
    console.log('Minte:', tx.hash);
    return tx.hash;
  } catch (err) {
    console.error('Erreur mint:', err.message);
    throw err;
  }
}

// Lire le solde on-chain
async function solde(address) {
  const bal = await krt.balanceOf(address);
  return ethers.formatEther(bal);
}

// Dans la route certification
router.post('/certifications', auth, async (req, res) => {
  const { moduleId, score, walletAddress } = req.body;
  if (score >= 15 && walletAddress) {
    await minterKRT(walletAddress, 5);
  }
  res.json({ certified: true, score });
});

Polygon pour Kora
Polygon = L2 compatible Ethereum. Meme Solidity, meme ethers.js. Fees de quelques centimes vs dizaines de $ sur Ethereum. MATIC sur Binance. Tests : Mumbai testnet (MATIC gratuit sur faucet). Pour Kora prod : Polygon est le bon choix.
