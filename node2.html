Modules — diviser le code en fichiers.

Au lieu de tout mettre dans index.js, tu divises en modules. module.exports expose ce que le fichier partage. require() importe ce dont tu as besoin.

Structure projet NodeUn projet bien structure : index.js (entree), routes/ (endpoints API), controllers/ (logique), models/ (schemas BDD), middleware/ (auth, validation), utils/ (fonctions partagees), config/ (configuration). Commence simple et structure progressivement.

ES Modules vs CommonJS
CommonJS (require) = standard historique. ES Modules (import/export) = standard moderne. React et Vue utilisent ES Modules. Pour activer dans Node : type: 'module' dans package.json. On garde CommonJS dans ce cours pour la compatibilite maximale avec les librairies.
Le module fs — lire et ecrire des fichiers.

fs est un module natif Node.js — zero npm install. Il lit, cree, modifie et supprime des fichiers. Utile pour les logs, configs JSON, exports de donnees.

fs sync vs asyncfs.readFileSync() bloque l'execution — simple mais bloque le serveur. fs.promises.readFile() est asynchrone — n'attend pas, n'bloque pas les autres requetes. En production toujours utiliser la version async.
Module fs — lecture et ecriture
const fs = require('fs');
const path = require('path');

// Lire (sync — pour les scripts simples)
const raw = fs.readFileSync('data.json', 'utf8');
const data = JSON.parse(raw);

// Lire (async — production)
async function lireData() {
  const raw = await fs.promises.readFile('data.json', 'utf8');
  return JSON.parse(raw);
}

// Ecrire
const portfolio = [{ ticker: 'BTC', qty: 0.5 }];
fs.writeFileSync('portfolio.json', JSON.stringify(portfolio, null, 2));

// Verifier existence
if (fs.existsSync('portfolio.json')) console.log('OK');

// Creer dossier
fs.mkdirSync('logs', { recursive: true });

// Ajouter (append)
fs.appendFileSync('app.log', new Date().toISOString() + '\n');
📄
path — eviter les bugs de chemins
path.join(__dirname, 'data', 'users.json') construit un chemin correct sur Windows et Linux. __dirname = dossier du fichier courant. Toujours utiliser path.join plutot que les concatenations — les slashes varient selon l'OS.
JSON comme BDD temporaire — prototyper rapidement.

Avant d'avoir une vraie BDD, on stocke les donnees dans des fichiers JSON. Parfait pour commencer et comprendre la logique CRUD.

JSON BDD temporairePour un prototype avec quelques utilisateurs, un fichier JSON suffit. Pour la production avec des milliers d'utilisateurs : PostgreSQL. L'avantage JSON : zero config, lisible, facile a debugger. L'inconvenient : pas de concurrence (deux requetes simultanees peuvent corrompre le fichier).
CRUD avec fichiers JSON
const fs = require('fs');
const path = require('path');
const FILE = path.join(__dirname, 'data', 'users.json');

function chargerUsers() {
  try {
    if (!fs.existsSync(FILE)) return [];
    return JSON.parse(fs.readFileSync(FILE, 'utf8'));
  } catch { return []; }
}

function sauvegarder(users) {
  fs.mkdirSync(path.dirname(FILE), { recursive: true });
  fs.writeFileSync(FILE, JSON.stringify(users, null, 2));
}

function creerUser(nom, email) {
  const users = chargerUsers();
  const user = { id: Date.now().toString(), nom, email, krt: 0 };
  users.push(user);
  sauvegarder(users);
  return user;
}

function trouver(email) {
  return chargerUsers().find(u => u.email === email);
}

const user = creerUser('Valentin', 'val@kora.fr');
console.log('Cree:', user);

Pourquoi pas localStorage
localStorage c'est cote navigateur. Le serveur Node.js n'y a pas acces. Les donnees serveur vivent sur le serveur — fichiers JSON, BDD, Redis. Cette separation est fondamentale : le serveur est la source de verite, le client affiche.
