Les fonctions.
Une fonction c'est un bloc de code reutilisable avec un nom. Tu l'appelles quand tu en as besoin. Les fonctions evitent la repetition, organisent le code, et rendent tout maintenable. C'est le concept le plus important de la programmation.

Une fonction — nommer et reutiliser un bloc de code.

Sans fonctions, si tu veux afficher le prix de 10 cryptos tu copies-colles le meme code 10 fois. Avec une fonction, tu ecris le code une fois et tu l'appelles 10 fois. Changer le comportement ? Tu modifies la fonction — une seule fois.

JavaScript — Declarer et appeler
// Declaration de fonction
function saluer() {
  console.log('Bonjour Kora !');
}

// Appeler la fonction
saluer(); // → 'Bonjour Kora !'
saluer(); // → 'Bonjour Kora !' (reutilisable)

// Fonction avec logique
function afficherPrix() {
  const btc = 65000;
  const eth = 3200;
  console.log(`BTC: ${btc} $ | ETH: ${eth} $`);
}

afficherPrix(); // → BTC: 65000 $ | ETH: 3200 $

// Hoisting — les declarations sont "hissees"
// Tu peux appeler une fonction avant sa declaration
direBonjour(); // Fonctionne !
function direBonjour() {
  console.log('Bonjour');
}
DRY — Don't Repeat YourselfC'est le principe fondamental de la programmation. Si tu copies-colles du code, c'est souvent le signe qu'il faudrait une fonction. Quand tu changes ce code copie, tu dois le changer a tous les endroits — et tu en oublies toujours un. Une fonction = un seul endroit a modifier.
⬆️
Le hoisting
Les declarations de fonctions (function nom(){}) sont "hissees" en haut de leur portee par le moteur JS. Ca signifie que tu peux appeler une fonction avant sa declaration dans le fichier. En revanche, les expressions de fonctions (const fn = function(){}) et les arrow functions (const fn = () => {}) ne sont PAS hissees — tu dois les declarer avant de les utiliser.
Parametres et return — l'entree et la sortie d'une fonction.

Les parametres sont les donnees qu'on passe a une fonction. return renvoie un resultat que tu peux stocker dans une variable. Une fonction sans return renvoie undefined.

JavaScript — Parametres et return
// Parametre = variable locale a la fonction
function saluer(prenom) {
  console.log(`Bonjour ${prenom} !`);
}
saluer('Valentin'); // → 'Bonjour Valentin !'
saluer('Kora');     // → 'Bonjour Kora !'

// Plusieurs parametres
function calculerValeur(prix, quantite) {
  return prix * quantite;
}
const valeurBTC = calculerValeur(65000, 0.5);
console.log(valeurBTC); // → 32500

// Valeurs par defaut
function afficherSignal(crypto = 'BTC', signal = 'HOLD') {
  return `${crypto} : ${signal}`;
}
console.log(afficherSignal());          // → 'BTC : HOLD'
console.log(afficherSignal('ETH'));      // → 'ETH : HOLD'
console.log(afficherSignal('SOL', 'BUY'));// → 'SOL : BUY'

return stoppe la fonction
Des qu'une fonction rencontre return, elle s'arrete et renvoie la valeur. Tout le code apres return dans la meme execution est ignore. C'est utile pour les "early returns" : if (condition invalide) return; en debut de fonction, puis le code normal. Ca evite l'imbrication excessive de if/else.
Arrow functions — la syntaxe moderne.

Les arrow functions (=>) sont une syntaxe plus courte pour ecrire des fonctions. Elles sont particulierement utiles pour les fonctions courtes et les callbacks. La syntaxe se simplifie encore plus pour les fonctions a une ligne.

JavaScript — Arrow functions
// Fonction classique
function doubler(n) {
  return n * 2;
}

// Arrow function equivalente
const doubler = (n) => {
  return n * 2;
};

// Une ligne → return implicite
const doubler = (n) => n * 2;

// Un seul parametre → parentheses optionnelles
const doubler = n => n * 2;

// Cas reels
const calculerFrais = (prix, taux = 0.001) => prix * taux;
const estEnHausse = (variation) => variation > 0;
const formaterPrix = (p) => `${p.toLocaleString('fr-FR')} $`;

console.log(calculerFrais(65000)); // → 65
console.log(estEnHausse(2.3));    // → true
console.log(formaterPrix(65000)); // → '65 000 $'
Quand utiliser les arrow functionsPour les fonctions courtes, les callbacks (map, filter, forEach), et les fonctions qui n'ont pas besoin de leur propre this. Les declarations classiques (function nom(){}) restent preferees pour les fonctions principales d'un module, notamment parce qu'elles profitent du hoisting.
Portee et closures — quelles variables sont accessibles ou.

La portee (scope) definit ou une variable est accessible. Les variables declarees dans une fonction ne sont pas accessibles dehors. Les closures permettent a une fonction interne d'acceder aux variables de la fonction externe meme apres que celle-ci a termine.

JavaScript — Portee et closures
// Portee globale vs locale
const cryptoGlobal = 'BTC'; // Accessible partout

function maFonction() {
  const prixLocal = 65000;  // Seulement dans cette fonction
  console.log(cryptoGlobal); // OK — accessible
  console.log(prixLocal);    // OK — locale
}
// console.log(prixLocal) → ReferenceError !

// Closure — fonction qui "capture" des variables
function creerCompteur(depart = 0) {
  let valeur = depart;

  return {
    incrementer: () => ++valeur,
    decrementer: () => --valeur,
    valeur: () => valeur
  };
}

const krt = creerCompteur(248);
krt.incrementer();
krt.incrementer();
krt.incrementer();
console.log(krt.valeur()); // → 251

Les closures partout
Les closures sont partout en JavaScript sans qu'on s'en rende compte. Chaque fois qu'un event listener (addEventListener) capture une variable, c'est une closure. Chaque setTimeout qui utilise une variable exterieure est une closure. Les hooks React (useState, useEffect) sont bases sur les closures. C'est l'un des concepts les plus puissants et les plus utilises de JS.
