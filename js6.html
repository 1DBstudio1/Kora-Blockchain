Les boucles.
Une boucle repete du code plusieurs fois. Pour chaque crypto du portfolio, pour chaque ligne d'un tableau, pour chaque element d'une reponse API — les boucles sont la facon dont les programmes traitent des listes de donnees.

La boucle for — repeter N fois avec un compteur.

La boucle for classique repete un bloc de code un nombre determine de fois. Elle utilise un compteur (i), une condition d'arret, et une incrementation. C'est la boucle la plus polyvalente.

JavaScript — for classique
// Syntaxe : for(init; condition; step)
for (let i = 0; i < 5; i++) {
  console.log(`Tour ${i}`);
}
// → Tour 0, Tour 1, Tour 2, Tour 3, Tour 4

// Parcourir un tableau avec index
const cryptos = ['BTC', 'ETH', 'SOL'];

for (let i = 0; i < cryptos.length; i++) {
  console.log(`${i}: ${cryptos[i]}`);
}
// → 0: BTC / 1: ETH / 2: SOL

// En sens inverse
for (let i = cryptos.length - 1; i >= 0; i--) {
  console.log(cryptos[i]);
}
// → SOL / ETH / BTC

// Sauter des elements (de 2 en 2)
for (let i = 0; i < 10; i += 2) {
  console.log(i); // 0, 2, 4, 6, 8
}

i commence toujours a 0Les index de tableaux commencent a 0. La condition est i < tableau.length (strict inferieur). Si tu mets i <= tableau.length, tu accederas a un element inexistant (undefined) sur le dernier tour. C'est l'erreur classique des debutants.
for...of et forEach — la boucle moderne pour les tableaux.

for...of itere directement sur les valeurs d'un tableau sans compteur. forEach est une methode de tableau qui fait pareil avec une fonction callback. Plus lisibles que le for classique quand tu n'as pas besoin de l'index.

JavaScript — for...of et forEach
const cryptos = ['BTC', 'ETH', 'SOL'];

// for...of — valeurs directement
for (const crypto of cryptos) {
  console.log(crypto);
}

// forEach — methode tableau avec callback
cryptos.forEach((crypto, index) => {
  console.log(`${index}: ${crypto}`);
});

// for...of avec objets (destructuring)
const portfolio = [
  { ticker: 'BTC', prix: 65000, qte: 0.5 },
  { ticker: 'ETH', prix: 3200,  qte: 2   },
];

for (const { ticker, prix, qte } of portfolio) {
  console.log(`${ticker}: ${prix * qte} $`);
}

// Strings aussi iterables
for (const lettre of 'KORA') {
  console.log(lettre); // K, O, R, A
}

forEach vs map vs for...of
forEach execute une action pour chaque element mais ne retourne rien. map transforme et retourne un nouveau tableau. for...of itere avec plus de flexibilite (break, continue, await). La regle : utilise map quand tu veux un nouveau tableau transforme, forEach pour les effets de bord (log, update DOM), for...of quand tu as besoin de break ou de async/await a l'interieur.
while et do...while — boucler jusqu'a une condition.

while boucle tant qu'une condition est vraie. do...while execute le bloc au moins une fois puis verifie la condition. Utiles quand tu ne sais pas a l'avance combien de fois boucler.

JavaScript — while et do...while
// while — repete tant que condition vraie
let krt = 248;

while (krt < 260) {
  console.log(`KRT: ${krt}`);
  krt += 3; // Important ! Sinon boucle infinie
}

// do...while — execute au moins une fois
let tentatives = 0;
do {
  console.log(`Tentative ${tentatives + 1}`);
  tentatives++;
} while (tentatives < 3);

// Cas reel : simuler des donnees
const historique = [];
let prix = 60000;

while (historique.length < 5) {
  // Variation aleatoire entre -5% et +5%
  const variation = (Math.random() * 10 - 5) / 100;
  prix *= (1 + variation);
  historique.push(Math.round(prix));
}
console.log(historique);
La boucle infinieSi la condition d'une boucle while ne devient jamais fausse, la boucle tourne a l'infini et bloque le navigateur. Toujours s'assurer que quelque chose dans la boucle fait progresser vers la condition de sortie. Si ton onglet se bloque, c'est probablement une boucle infinie — ferme l'onglet.
break, continue, for...in — controler le flux des boucles.

break stoppe la boucle completement. continue passe a l'iteration suivante. for...in itere sur les cles d'un objet. Ces outils donnent un controle precis sur l'execution.

JavaScript — break, continue, for...in
const cryptos = ['BTC', 'ETH', 'SOL', 'XRP', 'BNB'];

// break — stopper des qu'on trouve ce qu'on cherche
for (const c of cryptos) {
  if (c === 'SOL') {
    console.log('SOL trouve !');
    break; // Stoppe la boucle
  }
  console.log(c); // BTC, ETH (SOL non affiche)
}

// continue — sauter l'iteration actuelle
for (const c of cryptos) {
  if (c === 'ETH' || c === 'XRP') continue;
  console.log(c); // BTC, SOL, BNB (ETH et XRP sautees)
}

// for...in — cles d'un objet
const btc = { ticker: 'BTC', prix: 65000, variation: 2.3 };

for (const cle in btc) {
  console.log(`${cle}: ${btc[cle]}`);
}
// ticker: BTC / prix: 65000 / variation: 2.3
// Prefere Object.entries() + for...of en pratique
