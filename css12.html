Variables CSS.
Les variables CSS — aussi appelees custom properties — permettent de stocker des valeurs et de les reutiliser partout dans ton CSS. Changer la couleur principale de tout un site en modifiant une seule ligne. C'est la fondation d'un CSS maintenable.

Une variable CSS — definir une fois, utiliser partout.

Les custom properties CSS se definissent avec deux tirets (--nom) dans un selecteur, et se lisent avec var(--nom). Definies dans :root, elles sont accessibles dans tout le document.

Tu utilises des variables CSS depuis le debut de ce module sans le savoir — var(--bg), var(--text), var(--grad). Voila comment elles sont definies dans le code de Kora.

CSS — Variables CSS
/* Definir les variables dans :root = global */
:root {
  --bg: #FAFAFA;
  --text: #1A1A1A;
  --b1: #6B1F2A;
  --b2: #C4622D;
  --grad: linear-gradient(135deg, #6B1F2A, #C4622D, #E8956D);
  --radius: 12px;
  --police: 'DM Sans', sans-serif;
  --spacing: 16px;
}

/* Utiliser les variables */
body {
  background: var(--bg);
  color: var(--text);
  font-family: var(--police);
}

.btn {
  background: var(--grad);
  border-radius: var(--radius);
  padding: var(--spacing);
}

/* Valeur de fallback si la variable n'existe pas */
.el {
  color: var(--couleur-speciale, #6B1F2A);
  /* Si --couleur-speciale n'est pas definie, utilise #6B1F2A */
}

La vraie puissanceSi tu veux changer la couleur principale de tout ton site de #6B1F2A a #1A3A6B (bleu), tu changes juste --b1 dans :root. Tout le reste s'adapte automatiquement — les boutons, les titres, les bordures, les gradients. Sans variables, tu devrais faire un rechercher-remplacer global dans tous tes fichiers CSS.
🎨
Design tokens
Les grandes entreprises tech (Google, Airbnb, Shopify) utilisent un concept appele "design tokens" — des valeurs nommees pour les couleurs, espacements, et typographie. Les variables CSS c'est l'implementation native de ce concept. Quand tu definis --b1: #6B1F2A tu crees un design token. C'est la fondation de tout systeme de design professionnel.
Themes et dark mode — redefinir les variables selon le contexte.

La vraie puissance des variables CSS c'est de pouvoir les redefinir dans un contexte different. Tu definis les couleurs de base dans :root, et tu les surcharges dans [data-theme="dark"] ou via prefers-color-scheme pour le dark mode automatique.

CSS — Dark mode avec variables
/* Theme clair — defaut */
:root {
  --bg: #FAFAFA;
  --surface: #FFFFFF;
  --text: #1A1A1A;
  --muted: #737373;
  --border: #EFEFEF;
}

/* Theme sombre — via attribut HTML */
[data-theme="dark"] {
  --bg: #0A0A0A;
  --surface: #1A1A1A;
  --text: #F0F0F0;
  --muted: #737373;
  --border: #2A2A2A;
}

/* Dark mode automatique — preference systeme */
@media (prefers-color-scheme: dark) {
  :root {
    --bg: #0A0A0A;
    --text: #F0F0F0;
  }
}

/* Le reste du CSS n'a pas besoin de changer */
body { background: var(--bg); color: var(--text); }
.card { background: var(--surface); border: 1px solid var(--border); }

prefers-color-scheme
La media query prefers-color-scheme detecte si l'utilisateur a active le dark mode sur son appareil (iOS, macOS, Windows, Android). Sans variables CSS, implmenter le dark mode demande de dupliquer toutes tes regles CSS. Avec les variables, tu changes juste les valeurs dans :root — tout le reste s'adapte automatiquement.

Cree un systeme de variables CSS complet pour ta page profil. Toutes les couleurs, espacements et polices doivent etre des variables. Ajoute un theme dark mode.
Ton CSS doit avoir
  • Au moins 6 variables dans :root (couleurs, police, radius, spacing)
  • var() utilise pour toutes les couleurs — aucune couleur ecrite directement
  • Un selecteur [data-theme="dark"] ou @media prefers-color-scheme:dark
  • Les memes variables redefinies pour le dark mode
  • body et .card utilisant les variables pour background et color
