Le formulaire de connexion de Kora, c'est du HTML.

Chaque fois que tu tapes ton email et ton mot de passe quelque part, il y a une balise <form> derriere. La balise form regroupe tous les champs et definit ou et comment envoyer les donnees.

En HTML, un formulaire c'est simple : une balise <form> qui contient des champs <input>, des labels <label>, et un bouton <button> ou <input type="submit"> pour envoyer.

HTML — Formulaire de connexion basique
<form action="/login" method="post">

  <label for="email">Email</label>
  <input type="email" id="email" name="email"
         placeholder="votre@email.com" required>

  <label for="password">Mot de passe</label>
  <input type="password" id="password" name="password"
         required>

  <button type="submit">Se connecter</button>

</form>
Rendu dans le navigateur
action et methodL'attribut action dit ou envoyer les donnees (une URL de ton serveur). L'attribut method dit comment — GET (donnees dans l'URL, pour les recherches) ou POST (donnees dans le corps, pour les mots de passe et donnees sensibles). En HTML pur, sans backend, les formulaires ne "font" rien — c'est JavaScript ou un serveur qui traite les donnees.

La balise <input> a une dizaine de types differents.

C'est la balise la plus versatile du HTML. Selon l'attribut type, elle change completement de comportement — champ texte, email, mot de passe, case a cocher, bouton radio, nombre, date...

placeholder vs valueplaceholder est le texte gris qu'on voit dans le champ vide — il disparait quand l'utilisateur tape. C'est indicatif, pas une vraie valeur. value c'est la valeur par defaut deja remplie dans le champ que l'utilisateur peut modifier. required rend le champ obligatoire — le navigateur bloque l'envoi du formulaire si le champ est vide.
Trois autres elements indispensables.

En plus des inputs, tu as besoin de <textarea> pour les longs textes, <select> pour les menus deroulants, et <button> pour les actions.

HTML — textarea, select, button
<!-- Zone de texte multi-lignes -->
<textarea name="message" rows="4"
           placeholder="Votre message..."></textarea>

<!-- Menu deroulant -->
<select name="domaine">
  <option value="">Choisir un domaine</option>
  <option value="crypto">Crypto & Web3</option>
  <option value="trading">Trading</option>
  <option value="code">Code</option>
</select>

<!-- Boutons -->
<button type="submit">Envoyer</button>
<button type="reset">Effacer</button>
<button type="button">Action JS</button>

La barre de recherche de Google
La barre de recherche de Google — la plus utilisee du monde, des milliards de fois par jour — c'est un input type="text" dans un form avec method="get". Quand tu cherches "Bitcoin", le formulaire envoie les donnees a google.com/search?q=Bitcoin. Tu peux voir le ?q= dans l'URL — c'est la valeur de l'input qui a ete envoyee via GET.
Le label — le detail qui change tout pour l'accessibilite.

La balise <label> est souvent negligee mais elle est cruciale. Elle lie visuellement et semantiquement un texte descriptif a son champ. Quand c'est bien fait, cliquer sur le label active automatiquement le champ correspondant.

La connexion se fait via l'attribut for du label et l'id du champ — ils doivent etre identiques.

HTML — Labels corrects
<!-- BIEN — label lie au champ -->
<label for="username">Identifiant</label>
<input type="text" id="username" name="username">
<!-- Cliquer sur "Identifiant" active le champ -->

<!-- BIEN — label englobe le champ -->
<label>
  Identifiant
  <input type="text" name="username">
</label>

<!-- MAL — pas de lien label/champ -->
Identifiant
<input type="text">
<!-- Lecteurs d'ecran ne savent pas a quoi sert ce champ -->
Les attributs importantsrequired — champ obligatoire. disabled — champ desactive. readonly — visible mais non modifiable. autocomplete="off" — desactive la suggestion du navigateur. maxlength="100" — limite le nombre de caracteres. min et max — pour les inputs numeriques et dates. pattern — expression reguliere pour valider le format.
