Le DOM.
Le DOM (Document Object Model) c'est la representation du HTML en JavaScript. Tu peux lire, modifier, creer et supprimer des elements HTML depuis JS. C'est la connexion entre ton code et ce que l'utilisateur voit.

Selectionner des elements — document.querySelector.

Avant de modifier un element HTML, tu dois d'abord le selectionner. querySelector utilise la meme syntaxe que CSS — c'est le sélecteur le plus utilise. querySelectorAll retourne tous les elements correspondants.

JavaScript — Selectionner des elements
// querySelector — premier element correspondant
const titre = document.querySelector('h1');
const prix = document.querySelector('#prix-btc');
const card = document.querySelector('.card');
const btn = document.querySelector('.nav .btn-primary');

// querySelectorAll — tous les elements (NodeList)
const cards = document.querySelectorAll('.card');
const liens = document.querySelectorAll('a');

// Parcourir un NodeList
cards.forEach(card => {
  console.log(card.textContent);
});

// Anciennes methodes (toujours utilisees)
const el = document.getElementById('mon-id');
const els = document.getElementsByClassName('ma-classe');

// Si rien ne correspond → null (toujours verifier !)
const el = document.querySelector('#inexistant');
if (el) { /* faire quelque chose */ }
querySelector utilise la syntaxe CSSC'est le point le plus important : querySelector('#id'), querySelector('.classe'), querySelector('div > p'), querySelector('[data-type="crypto"]'). Toutes les combinaisons CSS fonctionnent. C'est pourquoi apprendre CSS avant JS est si important.
🌳
C'est quoi le DOM
Le DOM (Document Object Model) est une representation en arbre de ton HTML. Chaque balise est un "noeud". document est la racine. document.body est le body. Quand le navigateur charge ton HTML, il construit ce arbre en memoire. JS peut lire et modifier cet arbre, et le navigateur re-affiche automatiquement ce qui a change. C'est la magie des pages dynamiques.
Lire et modifier le contenu — textContent et innerHTML.

Une fois l'element selectionne, tu peux lire ou modifier son contenu avec textContent (texte pur) ou innerHTML (HTML). Pour les attributs, getAttribute et setAttribute. Pour le style inline, element.style.

JavaScript — Lire et modifier
const el = document.querySelector('#prix');

// Lire le contenu
console.log(el.textContent); // Texte pur
console.log(el.innerHTML);   // HTML avec balises

// Modifier le contenu
el.textContent = '65 000 $';
el.innerHTML = '<strong>65 000 $</strong> 🚀';

// Attributs
el.getAttribute('data-crypto');
el.setAttribute('data-prix', 65000);
el.removeAttribute('disabled');

// Style inline
el.style.color = '#28CA41';
el.style.fontSize = '24px';
el.style.display = 'none';

// Valeur d'un input
const input = document.querySelector('input');
console.log(input.value); // Lire la valeur
input.value = '65000';   // Modifier la valeur

textContent vs innerHTMLUtilise textContent pour le texte pur — c'est plus sur car il n'interprete pas le HTML (protection XSS). Utilise innerHTML seulement si tu dois inserer du vrai HTML. Ne jamais mettre innerHTML = donneeUtilisateur directement — c'est une faille de securite majeure (XSS). Utilise textContent pour les donnees venant des utilisateurs.
Creer et inserer des elements — construire le HTML depuis JS.

Tu peux creer des elements HTML depuis scratch avec createElement, leur ajouter du contenu et des attributs, puis les inserer dans la page. C'est comme ca qu'on genere dynamiquement des listes de cards, des lignes de tableau, des notifications.

JavaScript — Creer et inserer
// Creer un element
const card = document.createElement('div');
card.className = 'card';
card.textContent = 'Bitcoin';

// Inserer dans le DOM
const liste = document.querySelector('.liste');
liste.appendChild(card);      // A la fin
liste.prepend(card);          // Au debut
liste.insertAdjacentElement(  // Position precise
  'beforeend', card
);

// Methode moderne — innerHTML (pour HTML complexe)
const creerCard = (crypto) => `
  <div class="card">
    <h3>${crypto.ticker}</h3>
    <p>${crypto.prix} $</p>
  </div>
`;

const portfolio = [
  { ticker: 'BTC', prix: 65000 },
  { ticker: 'ETH', prix: 3200 },
];
const container = document.querySelector('#container');
container.innerHTML = portfolio.map(creerCard).join('');

// Supprimer un element
card.remove(); // Supprime l'element du DOM

⚡
innerHTML vs createElement
innerHTML est plus rapide a ecrire mais moins sur (risque XSS si les donnees viennent d'un utilisateur). createElement est plus verbeux mais securise et performant pour les grandes listes. La bonne pratique : innerHTML avec des template literals pour du HTML statique ou des donnees controlees, createElement pour les donnees utilisateur.
Manipuler les classes — classList, la bonne pratique.

Plutot que de modifier element.style directement (style inline), la bonne pratique est de manipuler les classes CSS avec classList. Tu gardes la separation HTML/CSS/JS et ton code est bien plus maintenable.

JavaScript — classList
const card = document.querySelector('.card');

// Ajouter une classe
card.classList.add('active');
card.classList.add('featured', 'highlight');

// Supprimer une classe
card.classList.remove('active');

// Toggle — ajoute si absente, supprime si presente
card.classList.toggle('dark');

// Verifier si une classe est presente
card.classList.contains('active'); // → true/false

// Remplacer une classe
card.classList.replace('hold', 'buy');

// En pratique — afficher/cacher un menu
const menu = document.querySelector('.menu');
const btn = document.querySelector('.burger');

btn.addEventListener('click', () => {
  menu.classList.toggle('open');
});
