Les operateurs — calculer et manipuler des valeurs.

Les operateurs arithmetiques (+, -, *, /, %) fonctionnent comme en mathematiques. Les operateurs d'assignation raccourcissent le code. Les operateurs logiques (&&, ||, !) combinent des conditions.

JavaScript — Operateurs
// Operateurs arithmetiques
const prix = 65000;
const quantite = 0.5;

prix + 1000    // → 66000 (addition)
prix - 5000    // → 60000 (soustraction)
prix * quantite // → 32500 (multiplication)
prix / 2        // → 32500 (division)
65003 % 1000   // → 3 (modulo = reste)
2 ** 10         // → 1024 (puissance)

// Operateurs d'assignation
let krt = 248;
krt += 5;   // krt = krt + 5 = 253
krt -= 10;  // krt = krt - 10 = 243
krt *= 2;   // krt = krt * 2 = 486
krt++;      // krt = krt + 1 (increment)
krt--;      // krt = krt - 1 (decrement)

// Operateurs logiques
true && true   // → true  (ET — les deux)
true || false  // → true  (OU — au moins un)
!true          // → false (NON — inverse)

Le modulo en trading
L'operateur modulo (%) retourne le reste d'une division. 65003 % 1000 = 3 (les centimes apres les milliers). En trading ca sert a calculer les niveaux de support sur des chiffres ronds, a faire des barres de progression (valeur % max), ou a alterner des elements (index % 2 === 0 pour les pairs). C'est aussi la base de la cryptographie — le chiffrement RSA repose entierement sur des modulos de grands nombres.
Comparaisons — === pas ==.

Les operateurs de comparaison retournent un booleen (true ou false). La regle la plus importante : utilise toujours === (egalite stricte) au lieu de == (egalite laxiste). == fait de la coercition de type et produit des resultats surprenants.

JavaScript — Comparaisons
// Comparaisons de base
65000 > 60000   // → true  (superieur)
65000 < 70000   // → true  (inferieur)
65000 >= 65000  // → true  (superieur ou egal)
65000 <= 64999  // → false (inferieur ou egal)

// === (strict) vs == (laxiste)
42 === 42     // → true  (meme valeur, meme type)
'42' === 42  // → false (string != number)
'42' == 42   // → true  (== convertit les types !)
0 == false   // → true  (bug potentiel !)
0 === false  // → false (correct)

// Inequalite
42 !== 43    // → true  (different strict)
42 != 43     // → true  (evite aussi)

// Combiner avec &&, ||
const prix = 65000;
prix > 60000 && prix < 70000  // → true (entre 60k et 70k)
prix < 50000 || prix > 80000  // → false (pas extremes)
Toujours === et !==Le double egal == fait de la coercition de type : il convertit les valeurs avant de comparer. Ca cree des bugs tres difficiles a debugger. La regle simple : utilise toujours === et !==. C'est la convention de tous les linters (ESLint) et des projets serieux.
if / else if / else — prendre des decisions.

if execute du code si une condition est vraie. else si elle est fausse. else if pour tester plusieurs conditions en cascade. C'est la structure de controle la plus fondamentale de la programmation.

JavaScript — if / else
const prix = 65000;
const objectif = 70000;
const stopLoss = 60000;

// if simple
if (prix >= objectif) {
  console.log('Objectif atteint — take profit');
}

// if / else
if (prix >= objectif) {
  console.log('Take profit');
} else {
  console.log('Position ouverte');
}

// if / else if / else
if (prix >= objectif) {
  console.log('Vendre — objectif atteint');
} else if (prix <= stopLoss) {
  console.log('Stop loss — couper la perte');
} else {
  console.log(`Prix : ${prix} — position maintenue`);
}

// Conditions multiples
const krt = 250;
const estActif = true;

if (krt >= 200 && estActif) {
  console.log('Eligible a la certification');
}

Truthy et Falsy
En JavaScript, les conditions ne doivent pas etre strictement true ou false. if(valeur) evalue valeur comme truthy ou falsy. Falsy : 0, '', null, undefined, NaN, false. Tout le reste est truthy. Exemple : if(krt) execute si krt n'est pas 0 ou null. if(nom) execute si nom n'est pas une string vide. C'est tres utilise pour tester si quelque chose "existe".
Ternaire et switch — des conditions plus concises.

L'operateur ternaire condense un if/else en une ligne. switch compare une valeur contre plusieurs cas. Les deux ont des situations ou ils sont plus lisibles que if/else.

JavaScript — Ternaire et switch
// Ternaire : condition ? siVrai : siFaux
const prix = 65000;
const signal = prix > 70000 ? 'SELL' : 'HOLD';
// Si prix > 70000 → 'SELL', sinon → 'HOLD'

// Dans un template literal
console.log(`Signal : ${prix > 70000 ? 'SELL' : 'HOLD'}`);

// Switch — comparer contre plusieurs cas
const crypto = 'BTC';

switch (crypto) {
  case 'BTC':
    console.log('Bitcoin — 65 000 $');
    break;
  case 'ETH':
    console.log('Ethereum — 3 200 $');
    break;
  case 'SOL':
    console.log('Solana — 170 $');
    break;
  default:
    console.log('Crypto inconnue');
}

// Nullish coalescing ?? — valeur par defaut
const prixAPI = null;
const prixAffiche = prixAPI ?? 'Chargement...';
// → 'Chargement...' (si prixAPI est null ou undefined)
Le ?? nullish coalescing?? retourne la valeur de droite seulement si la valeur de gauche est null ou undefined. Contrairement a ||, il ne se declenche pas pour 0 ou false. Parfait pour les valeurs par defaut : const krt = donneesUtilisateur.krt ?? 0. Si krt est 0 (valeur legitime), || retournerait 0 → faux. ?? retourne 0 correctement.
