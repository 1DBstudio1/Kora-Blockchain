Pseudo-classes et pseudo-elements.
Les pseudo-classes selectionnent des elements selon leur etat ou leur position. Les pseudo-elements selectionnent des parties virtuelles d'un element. Ensemble ils permettent des effets avances sans ajouter une seule ligne de HTML.

Les pseudo-classes d'etat — reagir aux interactions sans JavaScript.

:hover quand la souris est dessus, :focus quand l'element est selectionne au clavier, :active pendant le clic, :disabled quand desactive. Ces pseudo-classes permettent de creer des interfaces interactives en CSS pur.

:hover
Survol
Quand la souris est sur l'element. Changement de couleur, elevation, transition.
:focus
Focus clavier
Quand l'element est actif via clavier ou clic. Indispensable pour l'accessibilite.
:active
Clic
Pendant le clic. Effet d'appui sur les boutons.
:disabled
Desactive
Sur les inputs et boutons desactives. opacity + cursor:not-allowed.
CSS — Pseudo-classes d'etat
/* :hover — survol */
.btn:hover {
  opacity: 0.85;
  transform: translateY(-2px);
  box-shadow: 0 6px 20px rgba(107,31,42,0.3);
}

/* :focus — accessibilite clavier */
input:focus {
  outline: none;
  border-color: #C4622D;
  box-shadow: 0 0 0 3px rgba(196,98,45,0.15);
}

/* :active — pendant le clic */
.btn:active {
  transform: translateY(0);
  box-shadow: none;
}

/* :disabled — element desactive */
button:disabled {
  opacity: 0.4;
  cursor: not-allowed;
}

/* :checked — checkbox cochee */
input[type="checkbox"]:checked + label {
  color: #6B1F2A;
  font-weight: 500;
}
Demo interactive — survole et clique
Ne jamais supprimer outline sans remplaceroutline:none sur :focus supprime le contour de focus visible. C'est tentant visuellement mais catastrophique pour l'accessibilite — les utilisateurs clavier ne savent plus ou ils sont. Remplaces toujours outline par un box-shadow ou border-color visible. C'est ce que font tous les sites professionnels.
♿
L'accessibilite commence par :focus
Des millions d'utilisateurs naviguent sur le web uniquement avec le clavier — malvoyants, personnes avec troubles moteurs, power users. La pseudo-classe :focus est leur seul moyen de savoir ou ils se trouvent sur la page. Les WCAG (directives d'accessibilite du W3C) exigent que tout element interactif ait un style de focus visible. Google, Airbnb, Apple — tous ont des styles :focus soignes.
Pseudo-classes structurelles — cibler par position dans le DOM.

:nth-child, :first-child, :last-child, :not — ces pseudo-classes selectionnent des elements selon leur position dans leur parent. Parfaites pour les tableaux en zebra, les listes, les grilles.

CSS — Pseudo-classes structurelles
/* Premier et dernier */
li:first-child { font-weight: 500; color: #6B1F2A; }
li:last-child  { border-bottom: none; }

/* :nth-child — cibler par formule */
tr:nth-child(even) { background: #f5f5f5; } /* Pairs */
tr:nth-child(odd)  { background: white; }     /* Impairs */
li:nth-child(3)    { color: red; }             /* Le 3e */
li:nth-child(3n)   { color: blue; }            /* 3, 6, 9... */

/* :not — tous sauf */
li:not(:last-child) {
  border-bottom: 1px solid #EFEFEF;
}

/* :only-child — enfant unique */
p:only-child { margin: 0; }

/* :empty — element vide */
div:empty { display: none; }

nth-child en production
Les tableaux de donnees financieres sur Bloomberg, les listes de crypto sur CoinGecko, les tables de prix sur AWS — tous utilisent tr:nth-child(even) pour le zebra striping. Ca ameliore la lisibilite sans avoir a ajouter des classes CSS sur chaque ligne HTML. C'est ca l'elegance du CSS structural.
::before et ::after — du contenu virtuel sans toucher au HTML.

::before insere un element virtuel AVANT le contenu d'un element. ::after l'insere APRES. Ces pseudo-elements n'existent pas dans le HTML — ils sont crees par CSS. Parfaits pour les decorations, icones, overlays.

La propriete content est obligatoire — meme vide (content: ''). Sans content, le pseudo-element n'existe pas.

CSS — ::before et ::after
/* Ajouter du contenu textuel */
.required::after {
  content: ' *';
  color: red;
}

/* Ligne decorative avant un titre */
.section-title::before {
  content: '';              /* Vide mais obligatoire */
  display: block;
  width: 40px;
  height: 3px;
  background: linear-gradient(to right, #6B1F2A, #C4622D);
  margin-bottom: 10px;
  border-radius: 2px;
}

/* Overlay sur une image */
.card-image { position: relative; }
.card-image::after {
  content: '';
  position: absolute;
  inset: 0;               /* = top:0 right:0 bottom:0 left:0 */
  background: linear-gradient(to top, rgba(0,0,0,0.6), transparent);
}

/* Icone avec content */
.external-link::after {
  content: ' →';
  font-size: 12px;
  opacity: 0.6;
}
