Les evenements.
Les evenements c'est ce qui rend une page interactive. Clic, frappe clavier, soumission de formulaire, survol — chaque interaction utilisateur declenche un evenement que JS peut intercepter et traiter. C'est le coeur de l'interactivite web.

addEventListener — ecouter les interactions utilisateur.

addEventListener attache un "ecouteur" a un element. Quand l'evenement se produit, la fonction callback est appelee. C'est la methode standard — plus flexible que les attributs onclick dans le HTML.

JavaScript — addEventListener
const btn = document.querySelector('#mon-btn');

// Syntaxe de base
btn.addEventListener('click', function() {
  console.log('Clique !');
});

// Avec arrow function (recommande)
btn.addEventListener('click', () => {
  console.log('Clique !');
});

// Evenements courants
el.addEventListener('click', fn);       // Clic
el.addEventListener('dblclick', fn);    // Double clic
el.addEventListener('mouseenter', fn);  // Survol (entree)
el.addEventListener('mouseleave', fn);  // Survol (sortie)
inp.addEventListener('input', fn);      // Frappe dans input
inp.addEventListener('change', fn);     // Valeur changee
inp.addEventListener('keydown', fn);    // Touche pressee
form.addEventListener('submit', fn);    // Formulaire soumis
document.addEventListener('DOMContentLoaded', fn); // Page chargee

// Supprimer un ecouteur
btn.removeEventListener('click', maFonction);

En attente d'evenements...
addEventListener vs onclickonclick dans le HTML (onclick="maFonction()") ne peut avoir qu'un seul handler. addEventListener peut en avoir plusieurs sur le meme evenement. addEventListener permet aussi de supprimer le listener. En production, toujours utiliser addEventListener — jamais les attributs HTML.
L'objet event — les details de ce qui s'est passe.

Le callback d'un event listener recoit automatiquement un objet event (souvent appele e ou evt). Cet objet contient toutes les informations sur l'evenement — quelle touche a ete pressee, ou le clic a eu lieu, quel element a declenche l'evenement.

JavaScript — L'objet event
btn.addEventListener('click', (e) => {
  console.log(e.type);        // → 'click'
  console.log(e.target);      // L'element qui a recu le clic
  console.log(e.clientX);    // Position X du clic
  console.log(e.clientY);    // Position Y du clic
  console.log(e.timeStamp);  // Timestamp
});

// Evenement clavier
input.addEventListener('keydown', (e) => {
  console.log(e.key);      // 'Enter', 'a', 'Escape'...
  console.log(e.code);     // 'KeyA', 'Enter'...
  console.log(e.ctrlKey);  // true si Ctrl appuye
  console.log(e.shiftKey); // true si Shift appuye

  // Valider sur Enter
  if (e.key === 'Enter') soumettre();

  // Raccourci Ctrl+S
  if (e.ctrlKey && e.key === 's') {
    e.preventDefault(); // Bloquer le comportement par defaut
    sauvegarder();
  }
});

preventDefault et stopPropagation
e.preventDefault() empeche le comportement par defaut du navigateur. Sur un formulaire, ca empeche le rechargement de page. Sur un lien, ca empeche la navigation. Sur Ctrl+S, ca empeche la boite de dialogue de sauvegarde. e.stopPropagation() empeche l'evenement de "remonter" vers les elements parents (bubbling). Utilise l'un ou l'autre selon le besoin.
Evenements de formulaire — la validation en temps reel.

Les formulaires ont des evenements specifiques. submit pour la soumission, input pour chaque frappe, change pour quand la valeur change apres que l'utilisateur quitte le champ, focus et blur pour l'activation/desactivation.

JavaScript — Formulaires
const form = document.querySelector('form');
const input = document.querySelector('#montant');
const msg = document.querySelector('#message');

// Submit — toujours preventDefault sur les formulaires JS
form.addEventListener('submit', (e) => {
  e.preventDefault(); // Evite le rechargement de page
  const valeur = input.value.trim();
  if (!valeur) return; // Valider
  traiter(valeur);
  input.value = ''; // Vider le champ
});

// input — en temps reel, a chaque frappe
input.addEventListener('input', (e) => {
  const val = parseFloat(e.target.value);
  if (val < 0) {
    msg.textContent = 'Montant negatif !';
    msg.style.color = 'red';
  } else {
    msg.textContent = `Valeur : ${(val * 65000).toFixed(0)} $`;
    msg.style.color = 'green';
  }
});

// Entrer sur Enter dans un input
input.addEventListener('keydown', (e) => {
  if (e.key === 'Enter') form.dispatchEvent(new Event('submit'));
});

Delegation d'evenements — un seul listener pour tous les enfants.

Plutot que d'ajouter un listener sur chaque element d'une liste, tu ajoutes un seul listener sur le parent. Quand l'evenement remonte (bubbling), tu verifies quel enfant l'a declenche avec event.target. C'est plus performant et fonctionne meme pour les elements ajoutes dynamiquement.

JavaScript — Delegation d'evenements
// SANS delegation (a eviter) — 1 listener par element
document.querySelectorAll('.card').forEach(card => {
  card.addEventListener('click', () => {});
});
// Probleme : les nouvelles cards n'ont pas de listener !

// AVEC delegation — 1 listener sur le parent
const liste = document.querySelector('.liste');

liste.addEventListener('click', (e) => {
  // Verifier quel enfant a recu le clic
  const card = e.target.closest('.card');
  if (!card) return;

  const id = card.dataset.id;
  console.log(`Card ${id} cliquee`);
});

// closest — remonte jusqu'a trouver un ancetre matching
e.target.closest('.card'); // Ancetre .card le plus proche

// Gerer plusieurs types d'actions
liste.addEventListener('click', (e) => {
  if (e.target.matches('.btn-delete')) {
    e.target.closest('.item').remove();
  }
  if (e.target.matches('.btn-edit')) {
    ouvrirModal(e.target.closest('.item'));
  }
});
closest() — l'outil indispensableelement.closest('.selecteur') remonte dans le DOM jusqu'a trouver un ancetre correspondant au selecteur. Dans un listener delegue, si l'utilisateur clique sur un span a l'interieur d'un div.card, event.target sera le span. closest('.card') remontera jusqu'au div.card parent. C'est la cle de la delegation robuste.
🔼
Le bubbling — les evenements remontent
Quand tu cliques sur un element, l'evenement "remonte" vers le haut : element → parent → grand-parent → ... → document. C'est le "bubbling". La delegation exploite ce comportement — un listener sur le parent capte tous les clics sur ses enfants. C'est particulierement utile pour les listes generees dynamiquement : les nouveaux elements heritent automatiquement du listener parent.
