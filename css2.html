Les selecteurs.
Le selecteur c'est ce qui cible les elements HTML. Maitriser les selecteurs c'est maitriser le CSS — tu peux tout styler avec precision sans jamais toucher au HTML.

Le selecteur dit a qui s'applique le style.

Avant de definir comment quelque chose ressemble, CSS a besoin de savoir a QUOI appliquer ce style. C'est le role du selecteur.

Il existe trois types de selecteurs de base : le selecteur de balise, le selecteur de classe, et le selecteur d'id. Chacun a une specificite differente et un usage different.

p
Selecteur de balise
Cible TOUS les elements de ce type. p cible tous les paragraphes, h1 tous les titres principaux.
.classe
Selecteur de classe
Cible tous les elements avec class="nom". Peut etre utilise sur plusieurs elements differents.
#id
Selecteur d'ID
Cible l'element unique avec id="nom". Un ID doit etre unique sur la page.
*
Selecteur universel
Cible TOUS les elements. Utilise pour les resets globaux comme box-sizing.
CSS — Les 4 selecteurs de base
/* Balise — cible tous les h1 */
h1 { color: #6B1F2A; }

/* Classe — cible les elements avec class="card" */
.card { border-radius: 12px; padding: 20px; }

/* ID — cible l'element avec id="hero" */
#hero { height: 100vh; background: #1A1A2E; }

/* Universel — tous les elements */
* { box-sizing: border-box; }
Quand utiliser quoiSelecteur de balise pour les styles globaux (tous les p ont la meme police). Classe pour les styles reutilisables (plusieurs elements partagent le meme style). ID pour un element unique. En pratique : evite les IDs pour le style, prefere les classes.
Classes et IDs — la difference cruciale.

Une classe peut etre utilisee sur plusieurs elements. Un ID doit etre unique sur la page. En CSS, la classe s'ecrit avec un point (.classe) et l'ID avec un diese (#id).

La vraie puissance des classes : un element peut avoir plusieurs classes. class="card featured" applique les deux ensembles de styles. C'est comme ca que fonctionnent les frameworks CSS modernes.

HTML + CSS — Classes multiples
<!-- HTML : plusieurs classes sur un element -->
<div class="card">Bitcoin</div>
<div class="card featured">Kora</div>
<div class="card">Ethereum</div>

/* CSS : styles partages + variation */
.card {
  padding: 20px;
  border-radius: 12px;
  border: 1px solid #EFEFEF;
}
.featured {
  border-color: #C4622D;
  background: rgba(196,98,45,0.05);
}

Tailwind et Bootstrap
Les frameworks CSS les plus populaires comme Tailwind CSS et Bootstrap sont entierement bases sur les classes. class="flex items-center gap-4 p-4" en Tailwind applique 4 classes differentes. C'est exactement le systeme de classes multiples que tu viens d'apprendre, pousse a l'extreme.
Les combinateurs — cibler avec precision.

Les combinateurs permettent de cibler des elements en fonction de leur relation avec d'autres elements dans le HTML — parent, enfant, frere...

CSS — Les 4 combinateurs
/* Descendant (espace) — tous les a dans .nav */
.nav a {
  color: #737373;
  text-decoration: none;
}

/* Enfant direct (>) — seulement les li directs de ul */
ul > li {
  padding: 8px 0;
}

/* Frere adjacent (+) — le p juste apres h2 */
h2 + p {
  margin-top: 0;
  color: #737373;
}

/* Tous les freres (~) — tous les p apres h2 */
h2 ~ p {
  line-height: 1.8;
}
Le plus utilise : le descendantLe selecteur descendant (espace) est de loin le plus utilise. .nav a cible tous les liens dans la navigation. .card h2 cible tous les titres dans les cartes. Simple et puissant.
🎯
Ciblage chirurgical
Les combinateurs CSS permettent d'etre tres precis sans ajouter des classes partout dans le HTML. Au lieu de mettre class="nav-link" sur chaque lien de navigation, tu ecris juste .nav a en CSS. C'est plus propre et plus maintenable.
Les pseudo-classes — cibler selon l'etat.

Les pseudo-classes selectionnent des elements selon leur etat ou leur position — quand on le survole, quand il est actif, quand c'est le premier enfant de son parent...

CSS — Pseudo-classes essentielles
/* :hover — quand la souris est dessus */
a:hover {
  color: #C4622D;
  text-decoration: underline;
}
.btn:hover {
  opacity: 0.88;
  transform: translateY(-2px);
}

/* :focus — quand l'element est selectionne */
input:focus {
  border-color: #C4622D;
  outline: none;
  box-shadow: 0 0 0 3px rgba(196,98,45,0.15);
}

/* :nth-child — cibler par position */
li:first-child { font-weight: bold; }
li:last-child { border-bottom: none; }
tr:nth-child(even) { background: #f5f5f5; }

/* :not — tous sauf */
li:not(:last-child) { border-bottom: 1px solid #EFEFEF; }
Interactivite sans JavaScript:hover et :focus permettent de creer des interactions visuelles sans une seule ligne de JavaScript. Les menus deroulants, les animations de boutons, les formulaires qui changent de couleur — tout ca peut etre CSS pur.
🎯Selecteurs en pratiqueExercice
Ecris le CSS pour une page Kora en utilisant plusieurs types de selecteurs. L'objectif c'est de montrer que tu sais cibler precisement sans ajouter des classes partout.
Ton CSS doit utiliser
  • Un selecteur de balise (h1, p, nav ou autre)
  • Un selecteur de classe avec un point (.quelquechose)
  • Un selecteur combinateur (.parent a ou similaire)
  • Une pseudo-classe :hover ou :focus sur un lien ou bouton
