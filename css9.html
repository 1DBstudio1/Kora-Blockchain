CSS Grid.
Grid est le systeme de layout bidimensionnel de CSS. La ou Flexbox gere une ligne OU une colonne, Grid gere des lignes ET des colonnes simultanement. C'est le meilleur outil pour les layouts de page complets.

Grid — deux dimensions en meme temps.

Flexbox c'est super pour une rangee ou une colonne. Grid c'est pour les layouts complets — plusieurs rangees ET colonnes en meme temps. Tu definis les colonnes et les lignes, et les elements se placent dans les cellules.

Un grid container avec display:grid. Ses enfants sont automatiquement les grid items. Tu definis le nombre de colonnes avec grid-template-columns.

CSS — Activer Grid
.grid {
  display: grid;
  grid-template-columns: 1fr 1fr 1fr; /* 3 colonnes egales */
  gap: 16px;
}

/* Raccourci avec repeat() */
.grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr); /* Identique */
  gap: 16px;
}

/* Flexbox vs Grid — quand utiliser quoi */
/* Flexbox : composants (navbar, boutons, une rangee) */
/* Grid : layouts de page (header, sidebar, main, footer) */
📐
Grid et Flexbox — complementaires
Grid et Flexbox ne se remplacent pas — ils se completent. Dans un projet reel, tu utilises les deux : Grid pour la structure globale de la page (colonnes, zones), Flexbox pour aligner les elements a l'interieur de chaque zone. Le catalogue de formations de Kora Academy est un Grid auto-fit. L'interieur de chaque card est du Flexbox.
L'unite fr et auto-fit minmax — le layout responsive magique.

fr signifie fraction de l'espace disponible. 1fr 1fr 1fr = 3 parts egales. 2fr 1fr = deux tiers et un tiers. Tu peux mixer fr avec px pour des colonnes fixes et flexibles.

La recette auto-fit minmax est l'une des plus puissantes de CSS — elle cree un layout responsive sans aucune media query.

CSS — fr, auto-fit, minmax
/* L'unite fr — fractions de l'espace */
.grid { grid-template-columns: 1fr 1fr 1fr; } /* 3 egales */
.grid { grid-template-columns: 2fr 1fr; }     /* 2/3 et 1/3 */

/* Mixer px et fr */
.layout {
  grid-template-columns: 280px 1fr; /* Sidebar fixe + contenu */
}

/* LA recette magique — responsive SANS media query */
.cards {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(280px, 1fr));
  gap: 16px;
}
/* Sur grand ecran : 3-4 colonnes */
/* Sur mobile : 1 colonne */
/* AUCUNE media query ! */

repeat(auto-fit, minmax(280px, 1fr))La recette en detail : auto-fit = cree autant de colonnes que possible. minmax(280px, 1fr) = chaque colonne fait minimum 280px, maximum 1fr. Resultat : sur 900px de large, tu obtiens 3 colonnes de 300px. Sur 500px, 1 colonne de 500px. C'est exactement comme ca que fonctionne le catalogue de Kora Academy.
Placer les elements — grid-column et grid-row.

Par defaut, les grid items se placent automatiquement dans les cellules dans l'ordre. Mais tu peux contreler exactement ou chaque element se place et sur combien de colonnes ou lignes il s'etend.

CSS — Placer les elements dans le Grid
.grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
}

/* Prend toute la largeur (de colonne 1 a la derniere) */
.hero {
  grid-column: 1 / -1; /* -1 = derniere ligne/colonne */
}

/* S'etend sur 2 colonnes */
.featured {
  grid-column: span 2;
}

/* Position exacte */
.sidebar {
  grid-column: 3 / 4;  /* Colonne 3 */
  grid-row: 1 / 3;     /* Lignes 1 et 2 */
}
✨
-1 — la notation negative
Dans CSS Grid, -1 signifie "la derniere ligne ou colonne". grid-column: 1 / -1 signifie "de la premiere a la derniere colonne" — c'est la facon la plus simple de faire un element pleine largeur sans connaitre le nombre exact de colonnes. Tres utile pour les sections hero qui doivent toujours occuper toute la largeur.
grid-template-areas — nommer ses zones comme une maquette.

grid-template-areas permet de dessiner visuellement le layout directement dans le CSS avec des noms. C'est la facon la plus lisible et la plus maintenable de definir un layout complexe.

CSS — grid-template-areas
.page {
  display: grid;
  grid-template-columns: 240px 1fr;
  grid-template-rows: 56px 1fr 48px;
  grid-template-areas:
    "header  header"   /* Ligne 1 */
    "sidebar main"    /* Ligne 2 */
    "footer  footer"; /* Ligne 3 */
  min-height: 100vh;
  gap: 0;
}

/* Assigner chaque element a sa zone */
.header  { grid-area: header; }
.sidebar { grid-area: sidebar; }
.main    { grid-area: main; }
.footer  { grid-area: footer; }

/* Un point = cellule vide */
/* "header header" */
/* "sidebar ."     — pas de contenu en bas a droite */

Grid ou Flexbox — la regle simpleFlexbox pour les composants internes (navbar, liste de boutons, une card). Grid pour le layout global (structure de la page avec header, sidebar, main, footer). En pratique tu utilises les deux ensemble — Grid pour la grande structure, Flexbox pour aligner les elements dans chaque zone.
